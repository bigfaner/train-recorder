# Evaluate e2e Test Cases

## Description

Call `/eval-test-cases` skill to evaluate test-cases.md for downstream executability. This quality gate ensures test cases can directly drive `/gen-test-scripts` to produce runnable scripts.

This task runs in the **main session** (`mainSession: true`) because it needs to spawn doc-scorer and doc-reviser subagents, which requires main session context (subagents cannot spawn subagents).

## Reference Files

- `testing/test-cases.md` — Test case document (generated by T-test-1)
- `prd/prd-spec.md` — PRD spec (for traceability verification)
- `prd/prd-user-stories.md` — User stories (for completeness verification)

## Acceptance Criteria

- [ ] `testing/eval/report.md` exists with final score
- [ ] Final score >= 90 (target)
- [ ] No Step Actionability dimension score < 20

## Implementation Notes

1. Run `/eval-test-cases` skill with `--target 90 --iterations 6`
2. Review the final report
3. If target not reached after 6 iterations, mark this task as blocked and report to user
4. If T-test-1 was skipped, mark this task as skipped as well

## Main Session Instructions

This section is read by `run-tasks` dispatcher when `mainSession: true`. Execute exactly these steps:

1. **Invoke skill**: `Skill(skill="forge:eval-test-cases", args=FEATURE)`
