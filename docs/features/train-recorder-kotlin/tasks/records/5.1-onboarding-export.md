### Key Decisions
- Export uses kotlinx.serialization JsonElement builder to serialize all 20 DB tables into a versioned JSON schema (version 1)
- Date range filtering cascades: sessions filtered by date, then workout exercises/sets/feelings filtered by session IDs, body measurements filtered by date, other sport records filtered by date
- Import validates JSON schema (version + required keys) before processing, then regenerates all entity IDs via UUID mapping to avoid conflicts with existing data
- Import uses transactionWithResult for atomic operation - any failure during import rolls back all changes
- Onboarding flow uses 4-step wizard (Welcome -> Choose Plan -> Exercises -> Ready) with PlanTemplate data class containing exercise categories that map to seeded exercises
- Three plan templates: PPL (3-day push/pull/legs), Upper-Lower (2-day split), Full Body (1-day)
- ClearAllData updated to preserve exercise library - confirmation message now explicitly states this
- DataExportHelper provides pure validation/filtering functions: validateExportJson, isWithinDateRange, buildIdMapping, getExportDateRangePresets
