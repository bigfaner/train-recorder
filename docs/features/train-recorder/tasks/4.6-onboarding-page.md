# 4.6: Onboarding Page

## Description

Implement the first-time onboarding flow with welcome steps, template selection, and plan configuration.

## Reference Files

- prd/prd-spec.md §5.11.3 — First-time onboarding specs
- prd/prd-user-stories.md — US-18
- design/tech-design.md — OnboardingService interface

## Acceptance Criteria

- [ ] Welcome page: 3-4 intro steps explaining core concepts (plan → workout → record → progress)
- [ ] Template selection: 3 template cards (PPL, Upper/Lower, Full Body) with descriptions
- [ ] Plan configuration: adjust weights for template exercises before creation
- [ ] "完成" creates plan from template and navigates to calendar
- [ ] "跳过" skips onboarding entirely, navigates to calendar
- [ ] Onboarding only shows on first use (checks OnboardingService.isOnboardingComplete)
- [ ] Settings page "新手引导" link resets onboarding and navigates here
- [ ] Template exercises pre-filled with default increments and rest times

## User Stories

- US-18 AC1-AC4: Welcome flow, template recommendation, pre-filled exercises, settings re-view

## Implementation Notes

- Multi-step flow using horizontal scroll or stack navigator
- OnboardingService.getTemplates() provides template data
- OnboardingService.createPlanFromTemplate() handles plan creation
- OnboardingService.markOnboardingComplete() persists completion flag
- ResetOnboarding clears flag from settings, navigates to onboarding.tsx
