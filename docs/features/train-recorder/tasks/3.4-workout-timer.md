# 3.4: Workout Execution - Timer Panel UI

## Description

Implement the rest timer UI panel that slides in after set completion, with countdown display, adjustment controls, and background notification integration.

## Reference Files

- ui/ui-design.md — Component 2: 训练执行 (timer states, timer layout diagram)
- ui/prototype/workout.html — Timer prototype section
- prd/prd-spec.md §5.2.4 — Background timer reliability

## Acceptance Criteria

- [ ] Timer panel slides up from bottom after "完成本组" with large countdown display (72px)
- [ ] Circular progress indicator around timer
- [ ] "-30s", "跳过", "+30s" adjustment buttons
- [ ] Countdown reaches 0 → text turns red + vibration + sound alert
- [ ] "开始下一组" button appears on countdown completion
- [ ] Timer counts in foreground while other exercise cards dim
- [ ] System notification shows remaining time when app backgrounded ("组间休息 · 剩余 1:30")
- [ ] Tapping notification returns to workout page with "开始下一组" ready
- [ ] Lock screen shows notification on timer completion
- [ ] Phone call / long absence: shows "休息时间已过，准备好了就开始下一组" on return
- [ ] Force-close recovery: app reopens to workout page with timer state computed from timestamp
- [ ] Timer auto-cancels on workout exit

## User Stories

- US-2 AC2-AC4: Auto timer after set, vibration+sound on complete, skip timer
- US-11 AC1-AC6: Background timer, notification, lock screen, phone recovery, force-close recovery

## Implementation Notes

- Timer state from useTimer hook wrapping timerStore
- Foreground: Animated countdown with useAnimatedStyle for smooth transitions
- Background: expo-background-fetch + expo-notifications local notification
- Timer display font: SF Pro Display Bold 72px
- Vibration:expo-haptics or Vibration.vibrate([0, 500])
- Timer persistence via timerStore which calls TimerService.persistState()
