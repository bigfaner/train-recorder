# 3.2: Calendar Page (Tab 1)

## Description

Implement the training calendar page with month-view grid, training type indicators, day detail card, and training start flow.

## Reference Files

- ui/ui-design.md — Component 1: 训练日历 (layout, states, interactions, data binding)
- ui/prototype/calendar.html — Interactive prototype
- prd/prd-ui-functions.md — UF-1: 训练日历
- design/tech-design.md — CalendarComputer interface, useCalendar hook

## Acceptance Criteria

- [ ] Month-view grid displays all days with training type color dots (push=blue, pull=green, legs=orange)
- [ ] Month navigation with left/right arrows
- [ ] Clicking a date shows detail card at bottom with training info
- [ ] Future training days show plan preview + "开始训练" button
- [ ] Completed days show green checkmark + training summary
- [ ] Completed_partial days show semi-transparent checkmark + "部分完成" label
- [ ] Rest days show "今天没有训练安排" message
- [ ] Empty calendar (no plan) shows "创建你的第一个训练计划" prompt
- [ ] Training type filter tabs below month header (全部/推/拉/蹲/其他运动)
- [ ] "开始训练" navigates to workout page
- [ ] "记录其他运动" navigates to other-sport page
- [ ] Skipped days show grey dashed indicator
- [ ] Consecutive skip warning (≥3) displays alert

## User Stories

- US-1 AC4: Training day shows today's training, click to enter workout
- US-6 AC1-AC7: Calendar display, click details, skip/unskip, consecutive skip warning
- US-13 AC1: Past date without record shows "补录训练" option

## Implementation Notes

- Calendar data from useCalendar hook wrapping CalendarComputer.computeMonth()
- Training type colors from ui-design.md color palette
- Detail card slides up from bottom on date selection
- Filter tabs highlight matching dates, dim others
- Skip/long-press interactions via gesture handlers
