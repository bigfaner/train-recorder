# 3.7: Exercise Library + Detail Pages

## Description

Implement the exercise library page (browse by category with search) and exercise detail page (PR, progress chart, recent sessions). Also supports selection mode for plan editing.

## Reference Files

- ui/ui-design.md — Component 4: 动作库 (browse and selection layouts, custom exercise creation)
- ui/prototype/exercise-library.html, exercise-detail.html — Interactive prototypes
- prd/prd-ui-functions.md — UF-4: 动作库
- prd/prd-spec.md §5.5 — Exercise library specs

## Acceptance Criteria

- [ ] Browse mode: exercises grouped by category (7 categories from PRD §5.5)
- [ ] Each exercise card shows: name, increment (kg), default rest time
- [ ] Search bar filters exercises by name in real-time
- [ ] Category headers expand/collapse on tap
- [ ] Click exercise → push to detail page
- [ ] Detail page: basic info (name, category, increment, rest, total sessions), PR section (max weight, max volume with dates), progress chart (line chart with PR highlights), recent 5 sessions
- [ ] Selection mode: checkboxes on exercises, multi-select, "完成" button returns selected list to plan editor
- [ ] "+ 自定义动作" button: form with name, category, increment, rest time
- [ ] Custom exercises appear in "自定义" category
- [ ] Left-swipe custom exercises to delete (with in-use confirmation)
- [ ] Built-in exercises show "编辑" to modify increment/rest only
- [ ] Soft-deleted exercises hidden from library

## User Stories

- US-9 AC1-AC7: Full exercise library management (browse, custom, edit, detail, delete)
- US-4 AC1,AC5: Progress chart and PR display per exercise

## Implementation Notes

- Exercise data from Exercise repo grouped by category
- PR and history from useExerciseHistory hook
- Progress chart uses victory-native LineChart
- Selection mode activated when navigated from plan editor (route param or context flag)
- Custom exercise creation: is_custom=1, category='custom'
- Delete confirmation checks PlanExercise references
