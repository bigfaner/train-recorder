# 4.1: History Page (Tab 3)

## Description

Implement the history page with 4-segment view (history/progress/volume/PR) showing training records, progress charts, capacity trends, and personal records.

## Reference Files

- ui/ui-design.md — Component 5: 历史数据与进步分析
- ui/prototype/history.html — Interactive prototype
- prd/prd-ui-functions.md — UF-5: 历史数据与进步分析
- prd/prd-spec.md §5.6, §5.6.1, §5.6.2 — History, edit/delete, backlog specs

## Acceptance Criteria

- [ ] 4-segment control: 历史/进步/容量/PR with swipe animation
- [ ] History panel: training record cards sorted by date descending, showing date, type, exercises, volume, satisfaction
- [ ] PR badge (green) on exercises that hit personal records
- [ ] Progress panel: exercise selector dropdown + line chart (date vs weight) with PR points highlighted green
- [ ] Volume panel: bar chart (date vs total volume) with summary card (weekly/monthly)
- [ ] PR panel: per-exercise cards showing max weight + date + volume record
- [ ] Left-swipe history cards to show edit/delete actions
- [ ] Edit navigates to workout detail edit mode
- [ ] Delete shows confirmation dialog, removes record and recalculates PRs
- [ ] Training type filter on history panel
- [ ] Empty state: "完成你的第一次训练" prompt
- [ ] Chart supports pinch-zoom and pan for long time ranges

## User Stories

- US-4 AC1-AC7: Progress charts, PR tracking, type filter, capacity trends, zoom
- US-12 AC1-AC5: Edit/delete history records, PR rollback, feeling edit
- US-13 AC1-AC4: Backlog workout display with "补录" tag

## Implementation Notes

- Charts use victory-native (Skia-based) for LineChart and BarChart
- History data from WorkoutSession repo with exercises joined
- Progress data from WorkoutSet grouped by session_date for selected exercise
- PR data from PRTracker.getPRList()
- Edit/delete triggers ProgressiveOverload.recalculateChain for affected exercises
- Delete with PR triggers PRTracker.recalculatePR
