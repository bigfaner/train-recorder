# 4.5: Other Sport + Sport Editor Pages

## Description

Implement the other sport recording page with sport type selection, dynamic metric inputs, and custom sport type editor.

## Reference Files

- ui/ui-design.md — Component 7: 其他运动记录
- ui/prototype/other-sport.html — Interactive prototype
- prd/prd-ui-functions.md — UF-7: 其他运动记录
- prd/prd-spec.md §5.8 — Other sport specs

## Acceptance Criteria

- [ ] Sport type grid: 2×2 preset cards (游泳/跑步/骑行/瑜伽) + "自定义运动" button
- [ ] Selecting a sport type highlights it and shows dynamic input form below
- [ ] Dynamic form: input fields generated from SportMetric definitions for selected type
- [ ] Preset metrics: distance, time, laps, pace, heart rate, calories
- [ ] Custom sport editor (push page): name input + metric multi-select (preset + custom)
- [ ] Custom metric: name + unit inputs
- [ ] Save creates OtherSportRecord + SportMetricValue entries
- [ ] Custom sport types saved for reuse (auto-loads metrics on next selection)
- [ ] Note field for additional comments
- [ ] Date from calendar passed as default
- [ ] Back returns to calendar with updated day display

## User Stories

- US-7 AC1-AC6: Other sport recording, dynamic metrics, custom sports, calendar display, same-day dual recording, custom sport reuse

## Implementation Notes

- Sport types from SportType repo (presets + user custom)
- Metrics from SportMetric repo, loaded dynamically per selected sport_type_biz_key
- Custom sport: creates SportType (is_custom=1) + SportMetric entries
- Save: OtherSportRecord + array of SportMetricValue entries
- Calendar: CalendarComputer checks other_sport_records for day display
