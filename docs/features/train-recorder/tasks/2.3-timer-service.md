# 2.3: Timer Service (Foreground + Background)

## Description

Implement the rest timer service that runs reliably in foreground and background, using timestamp-based calculation and system local notifications for alerts.

## Reference Files

- design/tech-design.md — TimerService interface, PRD Coverage Map (US-2, US-11)
- prd/prd-spec.md §5.2.4 — Background timer reliability requirements

## Acceptance Criteria

- [ ] start(durationSeconds) begins countdown with startedAt = Date.now()
- [ ] Timer based on timestamp calculation: remaining = totalDuration - (now - startedAt)/1000
- [ ] pause()/resume() support for mid-timer pause
- [ ] skip() immediately completes the timer
- [ ] adjust(deltaSeconds) extends or shortens the remaining time
- [ ] persistState() saves timer state (startedAt, totalDuration, exerciseBizKey) to user_settings
- [ ] recoverState() restores timer from persisted state, computes elapsed time
- [ ] Background: expo-background-fetch keeps timer alive when app backgrounded
- [ ] Notification: expo-notifications schedules local notification on timer completion
- [ ] Force-close recovery: on app reopen, recoverState() computes from startedAt, shows "休息时间已过" if elapsed > totalDuration
- [ ] Unit tests for timer state transitions and recovery logic

## User Stories

- US-2 AC2-AC4: Auto-start timer after set completion, vibration on complete, skip support
- US-11 AC1-AC6: Background timer, notification, lock screen, phone call recovery, force-close recovery

## Implementation Notes

- Use Date.now() timestamps (not relative countdown) for reliability across app lifecycle
- expo-background-fetch for background execution + expo-notifications for alert
- persistState saves to user_settings with key 'active_timer_state' as JSON
- recoverState: read persisted state, compute remaining = totalDuration - (Date.now() - startedAt)/1000
- If remaining <= 0 on recovery: timer already completed, show "休息时间已过" alert
- onTick/onComplete callbacks for UI reactivity
