# Train Recorder — UI Functions

> Requirements layer: defines WHAT the UI must do. Not HOW it looks (that's ui-design.md).

## UI Scope

本功能需要以下 UI 界面：训练日历、训练执行、训练计划管理、动作库、历史分析、身体数据、其他运动记录、训练感受记录。

## UI Function 1: 训练日历

### Placement

- **Mode**: new-page
- **Target Page**: 首页/日历页
- **Position**: App 首页，用户打开 App 后的默认视图

### Description

日历视图展示训练安排和历史。每月视图，每个日期格子中标注训练类型（推/拉/蹲/其他运动/休息）。点击日期可进入当日训练或查看历史记录。

### User Interaction Flow

1. 用户打开 App → 显示当月日历，训练日用不同颜色标注训练类型
2. 点击有训练安排的未来日期 → 显示训练计划预览 + 「开始训练」按钮
3. 点击已完成的日期 → 显示该日训练详情摘要
4. 左右滑动 → 切换月份
5. 长按/拖动训练日 → 弹出调整日期的选项

### Data Requirements

| Field | Type | Source | Notes |
|-------|------|--------|-------|
| 日期 | date | 日历生成 | 每月所有日期 |
| 训练类型 | string | 训练计划排期 | 推/拉/蹲/休息/其他运动 |
| 训练状态 | string | 训练记录 | 已完成/待训练/已跳过 |
| 训练摘要 | string | 训练记录 | 如总容量、PR 等 |

### States

| State | Display | Trigger |
|-------|---------|---------|
| 空日历（无计划） | 提示「创建你的第一个训练计划」 | 首次使用，未创建计划 |
| 有排期的月份 | 各训练日显示类型标签和状态 | 有训练计划排期 |
| 有历史的日期 | 显示训练摘要（容量、PR 等） | 日期有已完成的训练记录 |

### Validation Rules

- 不可将训练日调整到过去的日期
- 不可在同一天安排多个力量训练日

## UI Function 2: 训练执行

### Placement

- **Mode**: new-page
- **Target Page**: 训练执行页
- **Position**: 从日历页点击「开始训练」进入

### Description

训练的核心操作页面。展示今日所有动作，逐组记录重量和次数，自动管理组间计时器。设计目标：单手操作，每组记录 ≤ 2 次点击。

### User Interaction Flow

1. 进入训练执行页 → 显示动作列表，每个动作卡片展示：名称、建议重量（可点修改）、目标组×次
2. 点击动作卡片展开 → 显示当前组的输入区域（重量预填充、次数输入）
3. 修改重量（可选）→ 点击次数输入 → 点击「完成本组」→ 保存并启动倒计时
4. 倒计时显示 → 大数字倒计时 + 跳过/延长按钮
5. 倒计时结束 → 振动提醒 + 「开始下一组」按钮
6. 完成所有组 → 动作卡片标记为「已完成」
7. 完成所有动作 → 进入感受记录页面

### Data Requirements

| Field | Type | Source | Notes |
|-------|------|--------|-------|
| 动作名称 | string | 训练计划 | 当前训练日的动作列表 |
| 建议重量 | number | 加重算法 | 基于上次表现计算 |
| 实际重量 | number | 用户输入 | 可覆盖建议值 |
| 实际次数 | number | 用户输入 | 本组实际完成的次数 |
| 目标次数 | number | 训练计划 | 用于判断是否达标 |
| 倒计时剩余 | number | 计时器 | 组间休息剩余秒数 |
| 已完成组数 | number | 实时计算 | x/总组数 |

### States

| State | Display | Trigger |
|-------|---------|---------|
| 未开始 | 所有动作卡片折叠，显示「开始训练」按钮 | 刚进入页面 |
| 进行中（记录组） | 当前动作展开，显示组输入区域 | 用户选中一个动作 |
| 进行中（组间休息） | 大倒计时显示，其他动作卡片变暗 | 一组记录完成 |
| 已完成 | 所在动作卡片标记 ✓，进度显示完成 | 所有组/动作完成 |
| 后台计时中 | 通知栏/锁屏显示倒计时剩余 | App 切到后台 |

### Validation Rules

- 重量必须 > 0
- 次数必须 ≥ 0（0 表示失败）
- 倒计时范围 30-600 秒

## UI Function 3: 训练计划管理

### Placement

- **Mode**: new-page
- **Target Page**: 训练计划管理页
- **Position**: 从设置或导航入口进入

### Description

创建、编辑和管理训练计划。支持固定周期和无限循环两种模式。定义每个训练日的分区和动作。

### User Interaction Flow

1. 点击「新建计划」 → 输入计划名称
2. 选择计划模式（固定周期 / 无限循环）
3. 如固定周期 → 输入周期长度（周数）
4. 添加训练日 → 选择训练类型（推/拉/蹲/自定义）→ 从动作库选择动作 → 设置目标组数和次数
5. 重复步骤 4 直到所有训练日定义完成
6. 保存计划 → App 自动排期到日历
7. 可随时编辑计划，编辑后提示是否重新排期

### Data Requirements

| Field | Type | Source | Notes |
|-------|------|--------|-------|
| 计划名称 | string | 用户输入 | 如「5/3/1 周期」 |
| 计划模式 | string | 用户选择 | fixed_cycle / infinite_loop |
| 周期长度 | number | 用户输入 | 仅固定周期模式，1-12 周 |
| 训练日列表 | array | 用户编辑 | 每日包含：类型、动作列表、组数×次数 |
| 动作详情 | object | 动作库 | 动作名、加重增量、默认休息 |

### States

| State | Display | Trigger |
|-------|---------|---------|
| 新建 | 空表单，引导输入 | 点击「新建计划」 |
| 编辑 | 预填充现有计划数据 | 点击编辑已有计划 |
| 多计划切换 | 计划列表，当前激活计划标记 | 用户有多个计划 |

### Validation Rules

- 计划名称不能为空
- 至少定义 1 个训练日
- 每个训练日至少包含 1 个动作
- 每个动作必须设置目标组数和次数

## UI Function 4: 动作库

### Placement

- **Mode**: new-page
- **Target Page**: 动作库页面
- **Position**: 从训练计划管理中选动作时打开，也可从设置中直接管理

### Description

内置常见力量举和辅助训练动作的分类列表，支持用户自定义动作。每个动作可配置独立的加重增量和默认休息时间。

### User Interaction Flow

1. 打开动作库 → 按分类展示动作列表
2. 点击动作 → 查看详情（名称、分类、加重增量、默认休息）
3. 点击「自定义动作」 → 输入名称、选择分类、设置增量和休息时间
4. 在训练计划创建时 → 从动作库中勾选动作添加到训练日

### Data Requirements

| Field | Type | Source | Notes |
|-------|------|--------|-------|
| 动作名称 | string | 预置/用户输入 | 唯一标识 |
| 分类 | string | 预置分类 | 核心力量举/上肢推/上肢拉/下肢辅助/其他/自定义 |
| 加重增量 | number | 预置默认/用户配置 | 单位 kg |
| 默认休息时间 | number | 预置默认/用户配置 | 单位秒 |
| 是否自定义 | boolean | 系统标记 | 区分预置和用户创建 |

### States

| State | Display | Trigger |
|-------|---------|---------|
| 浏览模式 | 分类列表，点击查看详情 | 从设置进入 |
| 选择模式 | 每个动作旁有勾选框 | 从计划创建中选择动作 |

### Validation Rules

- 动作名称不可重复
- 加重增量 > 0
- 休息时间 30-600 秒

## UI Function 5: 历史数据与进步分析

### Placement

- **Mode**: new-page
- **Target Page**: 历史分析页
- **Position**: 从导航入口进入

### Description

展示训练历史列表和进步趋势图表。支持按动作查看重量进步曲线、训练容量趋势、PR 追踪。

### User Interaction Flow

1. 进入历史页面 → 默认显示训练历史列表（按日期倒序）
2. 点击「进步曲线」标签 → 选择动作 → 显示重量随时间变化的折线图
3. 点击「容量趋势」标签 → 显示每次训练总容量的柱状图
4. 点击「PR 记录」标签 → 显示各动作的历史最高重量和最高容量
5. 点击列表中的某次训练 → 展示该次训练的详细数据

### Data Requirements

| Field | Type | Source | Notes |
|-------|------|--------|-------|
| 训练日期 | date | 训练记录 | X 轴 |
| 动作重量 | number | 训练记录 | Y 轴（进步曲线） |
| 训练容量 | number | 计算得出 | Σ(重量×次数)（容量趋势） |
| PR 值 | number | 历史最大值 | 最高重量、最高容量 |
| PR 日期 | date | 达成 PR 的训练日期 | PR 追踪 |

### States

| State | Display | Trigger |
|-------|---------|---------|
| 无数据 | 提示「完成你的第一次训练」 | 首次使用 |
| 历史列表 | 训练记录列表，按日期倒序 | 有训练记录 |
| 进步曲线 | 折线图 + PR 标注点 | 选择具体动作后 |
| 容量趋势 | 柱状图 | 切换到容量标签 |

### Validation Rules

- 进步曲线需要至少 2 个数据点
- PR 判断：单次重量超过历史最高即为新 PR

## UI Function 6: 身体数据记录

### Placement

- **Mode**: new-page
- **Target Page**: 身体数据页
- **Position**: 从导航入口进入

### Description

记录体重和围度数据，展示变化趋势图。可关联训练周期查看身体变化。

### User Interaction Flow

1. 进入身体数据页 → 显示「记录数据」按钮 + 最近一次身体数据
2. 点击「记录数据」 → 输入日期、体重、各围度 → 保存
3. 点击「趋势图」 → 选择指标 → 显示随时间变化的折线图

### Data Requirements

| Field | Type | Source | Notes |
|-------|------|--------|-------|
| 记录日期 | date | 用户选择 | 默认今天 |
| 体重 | number | 用户输入 | 精确到 0.1kg |
| 胸围 | number | 用户输入 | 精确到 0.1cm，可选 |
| 腰围 | number | 用户输入 | 精确到 0.1cm，可选 |
| 臂围 | number | 用户输入 | 精确到 0.1cm，可选 |
| 大腿围 | number | 用户输入 | 精确到 0.1cm，可选 |

### States

| State | Display | Trigger |
|-------|---------|---------|
| 无数据 | 提示「记录你的第一次身体数据」 | 首次使用 |
| 录入模式 | 各项输入字段 | 点击「记录数据」 |
| 趋势展示 | 折线图 + 数据列表 | 点击「趋势图」 |

### Validation Rules

- 体重为必填项，围度为选填
- 所有数值 > 0

## UI Function 7: 其他运动记录

### Placement

- **Mode**: new-page
- **Target Page**: 其他运动记录页
- **Position**: 从日历页点击「记录其他运动」进入

### Description

记录非力量训练的运动数据。支持自定义运动类型和记录指标。

### User Interaction Flow

1. 选择运动类型（预设或自定义）
2. 自定义运动类型时 → 输入名称 + 选择记录指标（距离/时间/配速/自定义）
3. 录入各项指标数据 → 保存
4. 可在历史中查看所有其他运动记录

### Data Requirements

| Field | Type | Source | Notes |
|-------|------|--------|-------|
| 运动类型 | string | 预设/自定义 | 如游泳、跑步 |
| 记录指标 | array | 用户配置 | 动态生成输入字段 |
| 指标值 | number | 用户输入 | 根据指标类型输入 |
| 日期 | date | 日历 | 记录日期 |
| 备注 | string | 用户输入 | 可选 |

### States

| State | Display | Trigger |
|-------|---------|---------|
| 选择运动类型 | 预设列表 + 「自定义」按钮 | 进入页面 |
| 录入数据 | 根据指标动态生成的输入字段 | 选择类型后 |
| 创建新类型 | 名称输入 + 指标多选 | 点击「自定义」 |

### Validation Rules

- 运动类型名称不能为空
- 至少选择 1 个记录指标
- 数值型指标必须 > 0

## UI Function 8: 训练感受记录

### Placement

- **Mode**: new-page
- **Target Page**: 训练感受页
- **Position**: 训练完成后自动进入

### Description

训练结束后的感受记录页面。记录整体疲劳度、满意度，以及各动作的独立感受。

### User Interaction Flow

1. 训练完成 → 自动进入感受记录页
2. 滑动设置整体疲劳度（1-10）
3. 滑动设置训练满意度（1-10）
4. 可选：为每个动作写感受备注
5. 可选：写整体备注
6. 点击「保存」 → 保存感受数据，更新加重建议 → 返回日历

### Data Requirements

| Field | Type | Source | Notes |
|-------|------|--------|-------|
| 疲劳度 | number(1-10) | 滑块 | 1=轻松, 10=筋疲力尽 |
| 满意度 | number(1-10) | 滑块 | 1=很差, 10=完美 |
| 各动作感受 | string | 文本输入 | 每个动作独立备注 |
| 整体备注 | string | 文本输入 | 自由文本 |

### States

| State | Display | Trigger |
|-------|---------|---------|
| 默认 | 疲劳度和满意度滑块在中间值（5），备注为空 | 训练完成后进入 |
| 保存确认 | 短暂显示「已保存」+ 加重建议更新提示 | 点击保存后 |

### Validation Rules

- 疲劳度和满意度有默认值，用户可直接保存
