# Task Record: 2.4 PR Tracker + Exercise History Services

Implemented PR Tracker service (pr-tracker.ts) and Exercise History service (exercise-history.ts). PR Tracker detects weight and volume PRs, supports recalculation for rollback on workout deletion, and provides Epley 1RM estimation. Exercise History provides recent session summaries, exercise detail summaries with PRs, and total session counts.

### Key Decisions

- PR types: 'weight' (max single-set weight) and 'volume' (max weight\*reps per set) tracked independently
- checkAndRecordPR returns the first PR type detected (weight checked before volume), but both are recorded in DB if both thresholds exceeded
- recalculatePR deletes all existing PRs for an exercise then rescans all workout_sets via workout_exercises to rebuild from scratch
- getEstimated1RM uses Epley formula: weight \* (1 + reps/30)
- Exercise History uses per-WorkoutExercise grouping to handle same-exercise-multiple-times (US-17)
- Skipped exercises are excluded from both PR calculations and session counts
- Used type aliases instead of empty interfaces for service type exports to satisfy @typescript-eslint/no-empty-object-type
