# 2.2: CalendarComputer Service

## Description

Implement the real-time calendar computation service that generates day-by-day training schedules from plan configuration and workout records, with no pre-generated schedule table.

## Reference Files

- design/tech-design.md — CalendarComputer interface, PRD Coverage Map (US-1, US-6)
- design/er-diagram.md — "日历排期计算" section in WorkoutSession notes
- prd/prd-spec.md §5.1, §5.9 — Calendar page and plan management specs

## Acceptance Criteria

- [ ] computeMonth(year, month, plan, days) returns CalendarDay[] for all days in the month
- [ ] weekly_fixed mode: maps weekday → training_day_biz_key from plan.weekly_config JSON
- [ ] fixed_interval mode: cycles through training days with rest_days gaps between
- [ ] Past dates with WorkoutSession show actual status (completed/completed_partial)
- [ ] Past dates without session but plan has training → dayType='training' (user may have skipped)
- [ ] skipTrainingDay(date) stores skipped date in user_settings JSON array
- [ ] unskipTrainingDay(date) removes from skipped list and restores assignment
- [ ] getConsecutiveSkips(planBizKey, beforeDate) returns consecutive skip count
- [ ] Other sport records displayed on correct dates
- [ ] Unit tests: weekly_fixed and fixed_interval scheduling, skip/unskip, consecutive skips

## User Stories

- US-1 AC2-AC4: Automatic scheduling (weekly_fixed and infinite_loop modes)
- US-6 AC1-AC7: Calendar display, drag-adjust, skip/unskip, consecutive skip warning
- US-13 AC1: Past date without record shows "补录训练" option

## Implementation Notes

- Skipped dates stored as JSON array in user*settings: key = `skipped_dates*{planBizKey}`
- fixed_interval: iterate training_days by order_index, insert rest_days rest days between each
- weekly_fixed: parse plan.weekly_config JSON (weekday number → training_day_biz_key mapping)
- CalendarDay.dayType enum: 'training' | 'rest' | 'other_sport' | 'completed' | 'completed_partial' | 'skipped'
- No Schedule table — all computed at query time from plan config + workout records + skip list
