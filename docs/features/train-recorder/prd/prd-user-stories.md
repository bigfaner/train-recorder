# User Stories: Train Recorder

## Story 1: 创建并执行周期化训练计划

**As a** 力量举训练者
**I want to** 创建一个推/拉/蹲分区的周期化训练计划，并在训练日按计划执行训练
**So that** 我的训练有系统化的安排，不再凭感觉决定每天练什么

**Acceptance Criteria:**
- Given 用户首次使用 App，When 用户点击「创建计划」，Then 进入计划创建流程
- Given 用户选择「无限循环」模式，When 用户定义了推日、拉日、蹲日三个训练日，Then App 自动按顺序排期到日历中
- Given 用户选择「固定周期」模式，When 用户设置了 4 周周期并定义了每周训练日，Then App 生成 4 周完整排期，第 5 周自动重新开始
- Given 训练日当天，When 用户打开 App，Then 日历上显示今日训练类型，点击可进入训练执行页面
- Given 用户已有正在执行的计划 A，When 用户创建并激活新计划 B，Then 日历清除计划 A 的排期，重新按计划 B 排期
- Given 用户创建计划时定义了 7 个训练日但未包含休息日，When 保存计划，Then 允许保存但提示「建议安排休息日」

## Story 2: 组间快速记录训练数据

**As a** 力量举训练者
**I want to** 在每组训练完成后快速记录重量和次数，并自动获得组间休息倒计时
**So that** 训练记录不打断训练节奏，同时保证休息时间充分

**Acceptance Criteria:**
- Given 用户在训练执行页面，When 进入一个动作，Then 显示建议重量（预填充）和目标组数×次数
- Given 用户完成一组，When 点击「完成本组」，Then 数据保存成功且自动启动组间倒计时（默认 180 秒）
- Given 组间倒计时进行中，When 倒计时到 0，Then 设备振动并播放提示音
- Given 组间倒计时进行中，When 用户点击「跳过」，Then 立即进入下一组记录
- Given 用户修改了预填充的建议重量，When 完成本组，Then 系统记录用户使用了自定义重量而非建议值
- Given 记录一组数据，When 操作完成，Then 总操作次数 ≤ 2 次点击
- Given 用户完成所有目标组数后想做额外组，When 点击「加一组」，Then 允许继续记录额外组，额外组不影响加重判断
- Given 用户在第 3 组（共 5 组）时退出训练页面，When 系统弹出确认，Then 用户确认后已完成的 2 组数据保留，未完成的动作标记为「未完成」
- Given 用户接到来电导致 App 进入后台，When 通话结束后返回 App，Then 训练页面恢复到中断前的状态，倒计时继续运行或已结束待提醒

## Story 3: 获得基于表现的渐进加重建议

**As a** 力量举训练者
**I want to** 在每次训练前获得每个动作的目标重量建议，并在训练完成后自动更新下次建议
**So that** 我不再需要纠结该加多重，渐进加重有据可依

**Acceptance Criteria:**
- Given 上次深蹲所有组均完成目标次数，When 本次训练加载深蹲，Then 建议重量 = 上次重量 + 深蹲的加重增量（如 5kg）
- Given 上次卧推有 1 组未完成目标次数，When 本次训练加载卧推，Then 建议重量 = 上次相同重量（不加不减）
- Given 卧推连续 2 次训练均有组未完成，When 本次训练加载卧推，Then 建议减重 10% 并提示用户
- Given 深蹲加重增量为 5kg，卧推加重增量为 2.5kg，When 两个动作均达标，Then 各自按自己的增量加重
- Given 建议的加重值，When 用户手动修改为目标重量，Then 使用用户修改的值，但下次建议仍基于实际完成情况计算
- Given 某动作从未训练过（无历史记录），When 加载该动作，Then 建议重量为空，提示用户输入初始重量，不强制要求
- Given 减重 10% 后的计算值不是整数（如 97.5 × 0.9 = 87.75），When 生成建议，Then 向下取整到最近的可用杠铃片组合（如 87.5kg）
- Given 用户连续 3 次训练均完成目标，When 计算加重建议，Then 显示提示「状态不错，考虑加大增量？」，但不自动修改增量

## Story 4: 查看进步趋势和个人记录

**As a** 力量举训练者
**I want to** 查看每个动作的历史进步曲线、训练容量趋势和个人记录（PR）
**So that** 我能看到自己的进步轨迹，确认训练计划是否有效

**Acceptance Criteria:**
- Given 用户有 ≥ 2 次深蹲训练记录，When 查看深蹲进步曲线，Then 显示以日期为 X 轴、重量为 Y 轴的折线图
- Given 用户打破了深蹲的历史最高重量，When 训练完成保存后，Then App 显示 PR 提醒「新个人记录！深蹲 140kg」
- Given 用户查看训练历史列表，When 按训练类型筛选「推日」，Then 只显示推日训练记录
- Given 用户查看容量趋势，When 选择月度视图，Then 显示该月每次训练的总容量柱状图
- Given 用户只有一个训练动作有历史数据，When 查看进步曲线，Then 正常显示该动作的曲线，无数据动作不显示
- Given 用户删除了某次训练记录（该记录包含 PR），When 刷新 PR 列表，Then PR 回退到上一次的历史最高值
- Given 用户查看进步曲线时间跨度超过 6 个月，When 数据点过多，Then 图表支持缩放和滑动浏览

## Story 5: 记录训练后的主观感受

**As a** 力量举训练者
**I want to** 在每次训练结束后记录疲劳度、满意度和各动作感受
**So that** 我的训练感受可追溯，帮助判断训练质量和恢复状态

**Acceptance Criteria:**
- Given 用户完成所有动作，When 进入感受记录页面，Then 显示疲劳度滑块（1-10）、满意度滑块（1-10）和各动作文本备注
- Given 用户记录疲劳度 ≥ 8 且满意度 ≤ 4，When 保存感受数据，Then 系统标记本次训练为「高疲劳低满意」，下次训练建议降低强度
- Given 用户未填写感受直接保存，When 保存成功，Then 使用疲劳度和满意度的默认值（5），文本备注为空
- Given 用户跳过了某些动作（训练中断），When 进入感受记录，Then 只显示已完成的动作的感受备注区
- Given 用户想补充修改上次训练的感受，When 在训练历史中找到该记录并点击编辑，Then 允许修改感受评分和备注，保存后更新

## Story 6: 在训练日历上管理训练安排

**As a** 力量举训练者
**I want to** 在日历视图上看到训练安排，并可以调整训练日期
**So that** 我能一目了然地知道哪天要练什么，临时有事时可以灵活调整

**Acceptance Criteria:**
- Given 训练计划已排期，When 用户打开日历页面，Then 每个训练日显示对应训练类型标签（推/拉/蹲）
- Given 用户周三有事，When 用户将周三的「拉日」拖动到周四，Then 周三变为休息日，周四变为拉日，后续排期自动顺延
- Given 用户点击已完成的训练日，When 该日有训练记录，Then 显示该日训练详情（动作、重量、容量、感受）
- Given 用户点击未来的训练日，When 该日有训练安排，Then 显示训练计划预览，可点击「开始训练」
- Given 用户想跳过今天的训练日，When 长按该日选择「跳过」，Then 该日标记为「已跳过」，该训练日的内容顺延到下一个可用日期
- Given 用户在日历上已跳过了 3 个连续训练日，When 第 4 个训练日到来时，Then 显示提示「已连续跳过 3 次训练，是否需要调整计划？」
- Given 用户已跳过的训练日，When 用户想恢复，Then 可以取消跳过状态，恢复原排期

## Story 7: 记录其他运动数据

**As a** 其他运动爱好者
**I want to** 在 App 中记录游泳等其他运动的数据（距离、时间等）
**So that** 我的所有运动数据集中在一个地方管理

**Acceptance Criteria:**
- Given 用户在日历上选择一个休息日，When 点击「记录其他运动」，Then 显示运动类型选择
- Given 用户选择「游泳」并配置了指标（距离、时间、趟数），When 进入记录页面，Then 显示对应的输入字段
- Given 用户首次记录某个运动类型，When 运动类型不在预设列表中，Then 用户可以自定义运动名称和指标
- Given 用户完成其他运动记录，When 保存成功，Then 日历上该日显示运动类型标签
- Given 用户在同一天已经完成了一次力量训练，When 用户还想记录游泳，Then 允许同一天既记录力量训练又记录其他运动，日历上两个标签并列显示
- Given 用户之前自定义了运动类型「登山」并配置了指标，When 用户再次选择「登山」，Then 自动加载之前的指标配置，无需重新设置

## Story 8: 追踪身体数据变化

**As a** 力量举训练者
**I want to** 定期记录体重和围度数据，并查看变化趋势
**So that** 我能了解身体组成的变化与训练效果的关联

**Acceptance Criteria:**
- Given 用户点击「记录身体数据」，When 进入录入页面，Then 显示日期（默认今天）、体重、胸围、腰围、臂围、大腿围输入框
- Given 用户有 ≥ 2 次体重记录，When 查看体重趋势图，Then 显示以日期为 X 轴、体重为 Y 轴的折线图
- Given 用户录入身体数据，When 只填写了体重未填围度，Then 只保存体重，未填写的围度不记录
- Given 用户录入过去某天的身体数据，When 选择了一个历史日期，Then 数据插入到正确的时间位置，趋势图按时间排序
- Given 用户想修改之前录入的身体数据，When 在历史记录中点击编辑，Then 允许修改，保存后趋势图自动更新

## Story 9: 管理动作库

**As a** 力量举训练者
**I want to** 从内置动作库中选择动作编制训练计划，也可以自定义新动作
**So that** 我可以快速构建计划，同时不被预设列表限制

**Acceptance Criteria:**
- Given 用户创建训练计划选择动作，When 打开动作库，Then 显示分类列表（核心力量举、上肢推、上肢拉、下肢、核心、肩部、自定义）
- Given 用户选择「深蹲」，When 添加到训练日，Then 使用深蹲的默认加重增量（5kg）和默认休息时间（180 秒）
- Given 用户需要的动作不在内置库中，When 点击「自定义动作」，Then 可以输入名称、选择分类、设置加重增量和默认休息时间
- Given 用户自定义了一个动作，When 下次创建计划时，Then 该动作出现在动作库的「自定义」分类中
- Given 用户想修改内置动作「深蹲」的默认加重增量，When 点击编辑，Then 允许修改增量（如从 5kg 改为 2.5kg），修改后的值覆盖默认值
- Given 用户删除了一个自定义动作，When 该动作已在某个训练计划中使用，Then 提示「该动作正在使用中，确认删除？」，确认后从计划中移除
- Given 用户想查看某个动作的详情，When 点击动作名称，Then 显示该动作的历史训练记录摘要（最近 5 次、PR、总训练次数）

---

## Story 10: 中途退出训练

**As a** 力量举训练者
**I want to** 在训练中途因故退出时，已记录的数据不丢失
**So that** 意外中断不会浪费我已经记录的训练数据

**Acceptance Criteria:**
- Given 用户完成了 3 个动作中的 2 个，When 用户点击返回或退出按钮，Then 弹出确认对话框「已完成 2/3 动作，确定结束训练？」
- Given 用户确认退出，When 保存数据，Then 已完成的 2 个动作数据正常保存，第 3 个动作标记为「未完成」
- Given 用户中途退出的训练，When 在日历上查看该日，Then 显示「已完成（部分）」状态（status: completed_partial）
- Given 用户中途退出的训练，When 加重算法计算下次建议，Then 已完成的动作正常参与加重判断，未完成的动作不纳入
- Given 用户中途退出时倒计时仍在运行，When 退出训练，Then 倒计时自动取消

## Story 11: 后台计时器可靠性

**As a** 力量举训练者
**I want to** 切换到其他 App 或锁屏后，组间倒计时仍然正确运行
**So that** 我可以在组间查看其他信息（如消息、音乐），不会错过下一组的时间

**Acceptance Criteria:**
- Given 用户在组间倒计时中切换到其他 App，When 倒计时剩余 90 秒，Then 计时器在后台继续倒计时
- Given 后台倒计时结束，When 用户在其他 App 或锁屏状态，Then 系统通知栏显示「休息结束，开始下一组！」
- Given 用户点击通知返回 App，When 训练页面恢复，Then 显示「开始下一组」按钮，倒计时显示 00:00
- Given 用户在倒计时中锁屏，When 倒计时结束，Then 锁屏界面显示提醒通知
- Given 用户在倒计时中接到电话，When 通话持续 5 分钟（远超倒计时），Then 通话结束后显示提醒「休息时间已过，准备好了就开始下一组」
- Given 用户强制关闭了 App，When 倒计时还在运行，Then 重新打开 App 后训练页面恢复，倒计时状态根据实际经过时间计算（已过则显示超时提示）

---

## Story 12: 编辑和删除历史训练记录

**As a** 力量举训练者
**I want to** 编辑或删除过去错误记录的训练数据
**So that** 历史数据准确反映实际训练情况，不影响加重建议

**Acceptance Criteria:**
- Given 用户查看训练历史，When 点击某次训练记录，Then 显示该次训练的详细数据，包含「编辑」和「删除」按钮
- Given 用户点击「编辑」，When 修改某组的重量或次数，Then 保存后加重建议基于修改后的数据重新计算
- Given 用户点击「删除」，When 确认删除，Then 该次训练记录及所有组数据被删除，加重建议回退到上一次训练的数据
- Given 用户删除了包含 PR 的训练记录，When 刷新 PR 列表，Then PR 回退到上一次的历史最高值
- Given 用户编辑了训练记录中的感受数据，When 保存后，Then 感受相关的训练建议（如降低强度）相应更新

---

## Story 13: 补录过去的训练记录

**As a** 力量举训练者
**I want to** 补录过去某天的训练数据（比如忘记记录时）
**So that** 训练历史完整，加重建议基于连续的数据

**Acceptance Criteria:**
- Given 用户在日历上选择一个过去的日子，When 该日没有训练记录，Then 显示「补录训练」选项
- Given 用户选择补录训练，When 选择训练类型和动作，Then 进入正常的训练记录流程（但无倒计时）
- Given 用户补录训练时输入的重量，When 保存成功，Then 该记录按日期插入到历史中，参与加重建议计算
- Given 用户补录的训练日期在最近一次训练之前，When 加重算法回溯，Then 中间的训练建议链会重新计算

---

## Story 14: 数据导出

**As a** 力量举训练者
**I want to** 导出我的训练数据到文件
**So that** 我可以备份数据或在电脑上进一步分析

**Acceptance Criteria:**
- Given 用户在设置中点击「导出数据」，When 选择导出范围（全部/最近 3 个月/最近半年），Then 生成包含训练记录、身体数据、其他运动记录的导出文件
- Given 导出完成，When 文件生成成功，Then 提供分享选项（邮件、云盘、本地保存）
- Given 导出的文件，When 用户在电脑上打开，Then 数据以结构化格式呈现，包含所有训练详情（日期、动作、组数、重量、次数、感受）

## Story 15: 重量单位切换

**As a** 力量举训练者
**I want to** 在公斤（kg）和磅（lbs）之间切换重量单位
**So that** 我可以用自己习惯的单位记录和查看数据

**Acceptance Criteria:**
- Given 用户当前使用 kg，When 在设置中切换为 lbs，Then 所有已记录的数据自动转换为 lbs 显示（1kg = 2.2046lbs）
- Given 用户切换为 lbs 后录入数据，When 保存，Then 数据以原始录入单位存储，显示时按当前设置单位转换
- Given 用户使用 lbs 设置，When 加重增量为 5lbs，Then 增量选项适应 lbs 单位（如 1/2.5/5/10 lbs）
- Given 用户使用磅，When 加重建议生成，Then 建议值以磅为单位显示，且取整到常用的杠铃片组合

---

## Story 16: 计划中的动作顺序调整

**As a** 力量举训练者
**I want to** 在训练执行过程中调整动作的顺序或跳过某个动作
**So that** 我可以根据健身房实际情况（器械被占用）灵活调整训练顺序

**Acceptance Criteria:**
- Given 用户在训练执行页面，When 长按某个未开始的动作卡片，Then 可以拖动调整顺序
- Given 用户想跳过某个动作，When 向左滑动动作卡片，Then 显示「跳过」选项，确认后该动作标记为「已跳过」
- Given 用户跳过了某个动作，When 训练结束，Then 跳过的动作不参与加重建议计算
- Given 用户跳过了某动作后又想补做，When 点击已跳过的动作，Then 可以取消跳过并开始记录

---

## Story 17: 同一动作在同一训练日出现多次

**As a** 力量举训练者
**I want to** 在一次训练中安排同一动作的多个变体（如深蹲 + 暂停深蹲）
**So that** 我的辅助动作和主动作可以是同一动作的不同变体

**Acceptance Criteria:**
- Given 用户在计划中添加了「深蹲」，When 再次添加「深蹲」，Then 允许添加，两次出现独立记录加重建议
- Given 用户想区分两次深蹲，When 添加动作时可以添加备注（如「暂停深蹲」），Then 训练页面显示备注区分
- Given 同一动作出现两次，When 加重建议计算，Then 两次独立计算，互不影响

---

## Story 18: 首次使用引导

**As a** 新用户
**I want to** 首次打开 App 时获得清晰的引导流程
**So that** 我可以快速上手，不需要摸索

**Acceptance Criteria:**
- Given 用户首次打开 App，When 没有任何训练数据，Then 显示简短的欢迎引导（3-4 步），介绍核心概念（计划→训练→记录→进步）
- Given 引导结束后，When 进入创建计划流程，Then 提供计划模板推荐（如「推/拉/蹲 3 日循环」），用户可选择模板快速创建或自定义
- Given 用户选择模板创建计划，When 进入动作配置，Then 动作已预填充常见动作和加重增量，用户只需调整重量即可
- Given 用户跳过引导直接使用，When 再次打开 App，Then 可在设置中找到「新手引导」重新查看
