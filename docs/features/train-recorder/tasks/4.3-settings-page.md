# 4.3: Settings Page (Tab 5)

## Description

Implement the settings page with training preferences, action library entry, notification toggles, and data management (export/import/clear).

## Reference Files

- prd/prd-ui-functions.md — UF-10: 设置
- ui/prototype/settings.html — Interactive prototype
- prd/prd-spec.md §5.11, §5.11.1, §5.11.2 — Settings, data export, unit switching specs

## Acceptance Criteria

- [ ] Grouped settings list: 训练设置 / 提醒 / 数据管理 / 关于
- [ ] Weight unit toggle: kg ↔ lbs cycle button with Toast confirmation
- [ ] Default rest time: bottom sheet with 90/120/180/240/300 second options
- [ ] Training reminder toggle (on/off)
- [ ] Vibration on rest end toggle (on/off)
- [ ] Sound on rest end toggle (on/off)
- [ ] "动作库管理" row navigates to exercise-library page
- [ ] "导出训练数据": bottom sheet with range selection (全部/3个月/6个月), generates JSON, share dialog
- [ ] "导入训练数据": bottom sheet with merge rules explanation, file picker, import progress
- [ ] "清除所有数据": warning bottom sheet, "不可撤销" message, secondary confirmation
- [ ] "新手引导" row navigates to onboarding page
- [ ] All toggles save immediately to UserSettings

## User Stories

- US-14 AC1-AC3: Data export with range selection, structured JSON format, share options
- US-15 AC1-AC4: Unit switching with global display update
- US-18 AC4: Settings link to re-view onboarding

## Implementation Notes

- Settings data from settingsStore (cached from UserSettings repo)
- Unit toggle: updates settingsStore.weightUnit, all display pages reactive
- Export: DataExportService.exportData(range) → shareFile()
- Import: DataImportService.validateFile → previewImport → importData
- Clear: delete all WorkoutSession/Feeling/PR data; keep Exercise library + UserSettings
- Toggle components from base UI
