# 2.4: PR Tracker + Exercise History Services

## Description

Implement the PR Tracker service (personal record detection and tracking) and Exercise History service (recent session summaries for exercise detail views).

## Reference Files

- design/tech-design.md — PRTracker interface, ExerciseHistoryService interface, PRD Coverage Map (US-4, US-9)
- prd/prd-spec.md §5.5, §5.6 — Exercise detail and history specs

## Acceptance Criteria

- [ ] checkAndRecordPR(exerciseBizKey, workoutSetBizKey, weight, reps) detects new PR and creates PersonalRecord
- [ ] PR types: 'weight' (max single-set weight) and 'volume' (max weight×reps)
- [ ] recalculatePR(exerciseBizKey) scans all WorkoutSets and rebuilds PR records from scratch
- [ ] getPRList() returns all PR records grouped by exercise
- [ ] getEstimated1RM(weight, reps) uses formula: weight × (1 + reps / 30)
- [ ] getExerciseSummary(exerciseBizKey) returns ExerciseDetailSummary with recent 5 sessions + PRs + total count
- [ ] getRecentSessions(exerciseBizKey, limit) returns session summaries with set details
- [ ] Deleting a workout with PR triggers recalculatePR → PR rolls back to previous max
- [ ] Unit tests: PR detection, rollback, 1RM estimation, session aggregation

## User Stories

- US-4 AC2,AC6: PR notification on new record, PR rollback on workout deletion
- US-9 AC7: Exercise detail with PR, progress summary, recent 5 sessions

## Implementation Notes

- checkAndRecordPR: compare against current max in personal_records table, create new record if exceeded
- recalculatePR: delete all PRs for exercise, rescan all workout_sets via workout_exercises, find new maxes
- getExerciseSummary: join workout_exercises + workout_sets, group by workout_session_biz_key, sort by session_date desc, limit 5
- 1RM estimation: Epley formula weight × (1 + reps/30), only accurate for reps ≤ 10
