# 3.6: Plan Management (Tab 2) + Editor Pages

## Description

Implement the plan management tab page (view mode), plan editor page (create/edit), and training day editor page with exercise selection and set configuration.

## Reference Files

- ui/ui-design.md — Component 3: 训练计划管理 (layout, states, interactions)
- ui/prototype/plan.html — Interactive prototype
- prd/prd-ui-functions.md — UF-3: 训练计划管理
- prd/prd-spec.md §5.9 — Training plan management specs

## Acceptance Criteria

- [ ] Plan tab shows active plan with training day cards (day name, exercise list)
- [ ] "新建计划" button to enter plan editor
- [ ] Plan editor: plan name input, mode selector (无限循环/固定周期), schedule mode (每周固定日/固定间隔)
- [ ] weekly_fixed: weekday toggle buttons, assign training type per weekday
- [ ] fixed_interval: rest days stepper (0-6), training day order list
- [ ] Training day editor: training type selector (推/拉/蹲/自定义), exercise list with add/remove
- [ ] Per-exercise: mode toggle (固定/自定义), fixed mode = sets×reps×weight inputs, custom mode = per-set weight×reps
- [ ] "从动作库添加" opens exercise library in selection mode (modal)
- [ ] Drag to reorder exercises within a training day
- [ ] Left-swipe to delete exercise from training day
- [ ] Save plan: validates name, at least 1 training day, each day has ≥1 exercise
- [ ] Switching active plan deactivates previous plan and re-schedules calendar
- [ ] Empty state: "创建你的第一个训练计划" prompt

## User Stories

- US-1 AC1-AC6: Plan creation, mode selection, schedule config, activation switching
- US-9 AC2: Default increment and rest time from exercise library
- US-17 AC1-AC2: Same exercise added multiple times with notes

## Implementation Notes

- Plan tab is view mode; edit mode is push page (plan-editor.tsx)
- Training day editor is a sub-push page (training-day-editor.tsx)
- PlanExercise.sets_config JSON: fixed mode {mode, target_reps, target_weight, target_repeat}, custom mode {mode, sets: [{target_reps, target_weight}]}
- Schedule validation: warn if training_days.length >= 7 with no rest days, but allow save
- plan_mode='fixed_cycle' shows cycle_length input; 'infinite_loop' hides it
