### Key Decisions
- TimerServiceImpl uses timestamp-based calculation (start_timestamp + total_duration_seconds - now) for accurate remaining time on app resume
- TimerNotificationHelper interface abstracts platform-specific notification and background execution (Android ForegroundService / iOS BackgroundTask), keeping core logic in commonMain
- TimerClock interface provides testability by wrapping Clock.System, allowing FakeClock in tests
- cancelInternal() is a silent cancel (no notification calls) used when switching timers; cancelTimer() is the public method that triggers notification cleanup only when a timer was active
- Ticker coroutine runs on a configurable CoroutineScope (default Dispatchers.IO), enabling test injection
- Resume from persisted state reads selectRunningTimerState from DB, calculates remaining, and either resumes countdown or triggers expiry alert
