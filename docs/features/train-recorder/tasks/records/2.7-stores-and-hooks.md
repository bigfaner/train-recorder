### Key Decisions
- Stores are factory functions (createWorkoutStore, createTimerStore, createSettingsStore) accepting dependency objects for testability
- Hooks accept store instances as parameters rather than importing singleton stores, enabling test injection
- useCalendar, useProgressiveOverload, useExerciseHistory use React useState for local async state since they wrap services that don't have Zustand stores
- useUnitConversion reads from settingsStore via useStore selector for reactive unit switching
- computeProgress and formatDisplayTime exported as pure functions for direct unit testing without renderHook
- workoutStore.recordSet triggers ProgressiveOverload recording, PR checking, and timer start in sequence with non-critical failure isolation
