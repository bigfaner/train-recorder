### Key Decisions

- Created DatabaseAdapter interface to abstract expo-sqlite sync API, enabling sql.js-based integration testing without native dependencies
- Used factory functions (createXxxRepo) instead of classes for simpler DI and testability
- bigint biz_key values are converted to number via toDbValue() at the SQLite boundary since both expo-sqlite and sql.js store INTEGER as JS number
- BaseRepository.create() excludes 'id' column from INSERT (auto-increment), includes it in SELECT
- Exercise soft-delete via is_deleted=1 flag; findAllActive() and findByCategory() filter is_deleted=0
- TrainingPlan.activatePlan() deactivates all other plans first (single active plan constraint)
- PlanExercise stores sets_config as JSON string; parse on read, stringify on write (application responsibility)
