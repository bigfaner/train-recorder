### Key Decisions
- Timestamp-based calculation (Date.now()) instead of relative countdown for reliability across app lifecycle
- TimerPhase internal state machine: idle -> running -> paused -> idle for clear state transitions
- On pause, store pausedAtRemaining; on resume, recalculate startedAt so timestamp math stays consistent
- persistState uses JSON serialization with BigInt as string for user_settings storage under key 'active_timer_state'
- recoverState computes elapsed time from persisted startedAt, fires onComplete if timer already expired (rest time has passed)
- adjust() recalculates startedAt and totalDuration to maintain timestamp-based countdown integrity
