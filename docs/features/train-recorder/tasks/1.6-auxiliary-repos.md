# 1.6: Auxiliary Repositories (Feeling, PR, Body, Sport, Settings)

## Description

Implement the remaining 9 repositories: Feeling, ExerciseFeeling, PersonalRecord, BodyMeasurement, OtherSportRecord, SportType, SportMetric, SportMetricValue, and UserSettings.

## Reference Files

- design/tech-design.md — Data Models Field Quick Reference
- design/er-diagram.md — All auxiliary entity details
- design/schema.sql — All auxiliary table definitions

## Acceptance Criteria

- [ ] feeling.repo.ts implements CRUD with findByWorkoutSessionBizKey
- [ ] exercise-feeling.repo.ts implements CRUD with findByFeelingBizKey, findByWorkoutExerciseBizKey
- [ ] personal-record.repo.ts implements CRUD with findByExerciseBizKey, findByType (weight/volume), findMaxByExercise
- [ ] body-measurement.repo.ts implements CRUD with findByDateRange, findLatest
- [ ] other-sport.repo.ts implements CRUD with findByDate, findBySportTypeBizKey
- [ ] sport-type.repo.ts implements CRUD with findAllIncludingCustom
- [ ] sport-metric.repo.ts implements CRUD with findBySportTypeBizKey
- [ ] sport-metric-value.repo.ts implements CRUD with findBySportRecordBizKey
- [ ] user-settings.repo.ts implements get/setValue pattern (upsert by setting_key)
- [ ] Integration tests with in-memory SQLite pass for all CRUD operations

## User Stories

- US-5 AC1-AC5: Feeling recording and editing (Feeling + ExerciseFeeling repos)
- US-4 AC2,AC6: PR tracking and rollback (PersonalRecord repo)
- US-8 AC1-AC5: Body data recording and trends (BodyMeasurement repo)
- US-7 AC1-AC6: Other sport recording (OtherSportRecord + SportType + SportMetric repos)
- US-15 AC1-AC4: Unit settings (UserSettings repo)

## Implementation Notes

- UserSettings uses get/setValue pattern: findByKey(key) → exists ? update : create
- PersonalRecord: pr_type is 'weight' or 'volume'; findMaxByExercise returns highest pr_value for given type
- BodyMeasurement: nullable circumference fields; findLatest returns most recent record_date entry
- SportMetricValue: composite reference (sport_record_biz_key + sport_metric_biz_key)
