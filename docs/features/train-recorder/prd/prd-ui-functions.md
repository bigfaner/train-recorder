# Train Recorder — UI Functions

> Requirements layer: defines WHAT the UI must do. Not HOW it looks (that's ui-design.md).

## Navigation Structure

底部 5 个 Tab 全局导航，所有主页面共享：日历、计划、历史、统计、设置。

| Tab  | 页面          | 说明                            |
| ---- | ------------- | ------------------------------- |
| 日历 | calendar.html | App 默认首页                    |
| 计划 | plan.html     | 训练计划管理，查看/编辑模式切换 |
| 历史 | history.html  | 训练记录与进步分析，4 个子面板  |
| 统计 | stats.html    | 宏观训练概览仪表盘              |
| 设置 | settings.html | 训练偏好、动作库入口、数据管理  |

非 Tab 页面（Push 页面，有返回按钮）：

| 页面                            | 入口                   |
| ------------------------------- | ---------------------- |
| 训练执行 (workout.html)         | 日历页「开始训练」按钮 |
| 训练感受 (feeling.html)         | 训练完成后自动进入     |
| 其他运动 (other-sport.html)     | 日历页入口             |
| 动作详情 (exercise-detail.html) | 动作库点击动作进入     |
| 动作库 (exercise-library.html)  | 设置页「动作库」入口   |

## UI Scope

本功能需要以下 UI 界面：训练日历、训练执行、训练计划管理、动作库（含动作详情）、历史分析、统计概览、身体数据、其他运动记录、训练感受记录、设置。

## UI Function 1: 训练日历

### Placement

- **Mode**: tab-page（Tab 1）
- **Target Page**: 日历页
- **Position**: App 默认首页

### Description

月视图日历展示训练安排和历史。每个日期格子下方用彩色圆点标注训练类型（推=蓝/拉=绿/蹲=橙）。点击日期在日历下方弹出详情卡片，显示当日训练信息。已完成日期右上角显示绿色勾选标记。

### User Interaction Flow

1. 用户打开 App → 显示当月日历，训练日用彩色圆点标注类型
2. 点击任意日期 → 日历下方弹出详情卡片
   - 有训练安排的未来日期：显示训练类型标签、动作列表、「开始训练」按钮
   - 已完成的日期：显示训练类型标签、动作列表、「已完成」标记
   - 休息日：显示休息标签 + "今天没有训练安排"
3. 点击「开始训练」→ 进入训练执行页
4. 左右切换月份（当前原型仅支持当月）
5. 点击月份导航下方的训练类型标签（全部/推/拉/蹲/其他运动）→ 高亮显示对应类型的日期，其他日期变暗

### Data Requirements

| Field        | Type   | Source       | Notes                  |
| ------------ | ------ | ------------ | ---------------------- |
| 日期         | date   | 日历生成     | 每月所有日期           |
| 训练类型     | string | 训练计划排期 | 推/拉/蹲/休息/其他运动 |
| 训练状态     | string | 训练记录     | 已完成/待训练/已跳过   |
| 训练动作摘要 | string | 训练计划     | 如"卧推/推举/三头"     |
| 详情日期文本 | string | 计算         | 如"5月7日 周四"        |
| 筛选类型     | string | 用户选择     | 当前筛选的训练类型标签 |

### States

| State                | Display                                                    | Trigger                    |
| -------------------- | ---------------------------------------------------------- | -------------------------- |
| 空日历（无计划）     | 提示「创建你的第一个训练计划」                             | 首次使用，未创建计划       |
| 有排期的月份         | 各训练日显示类型色点和状态                                 | 有训练计划排期             |
| 选中训练日           | 底部弹出详情卡片，显示动作和操作按钮                       | 点击有训练安排的日期       |
| 选中休息日           | 底部弹出详情卡片，显示休息提示                             | 点击无训练安排的日期       |
| 已完成训练日         | 日期右上角绿色勾选标记                                     | 日期有已完成的训练记录     |
| 已完成（部分）训练日 | 日期右上角显示半透明勾选标记，详情卡片显示「部分完成」标签 | 日期有未全部完成的训练记录 |

### Validation Rules

- 不可将训练日调整到过去的日期
- 不可在同一天安排多个力量训练日

## UI Function 2: 训练执行

### Placement

- **Mode**: push-page
- **Target Page**: 训练执行页
- **Position**: 从日历页点击「开始训练」进入，左上角返回日历

### Description

训练的核心操作页面。展示今日所有动作，逐组记录重量和次数，自动管理组间计时器。设计目标：单手操作，每组记录 ≤ 2 次点击。

### User Interaction Flow

1. 进入训练执行页 → 显示动作列表，每个动作卡片展示：名称、建议重量（含加重建议标签）、目标组×次
2. 点击动作卡片展开 → 显示当前组的输入区域（重量预填充、次数输入）
3. 修改重量（可选）→ 点击次数输入 → 点击「完成本组」→ 保存并启动倒计时
4. 倒计时显示 → 大数字倒计时 + 跳过/延长按钮
5. 倒计时结束 → 振动提醒 + 「开始下一组」按钮
6. 完成所有组 → 动作卡片标记为「已完成」
7. 完成所有动作 → 进入感受记录页面

### Data Requirements

| Field      | Type   | Source   | Notes                              |
| ---------- | ------ | -------- | ---------------------------------- |
| 动作名称   | string | 训练计划 | 当前训练日的动作列表               |
| 建议重量   | number | 加重算法 | 基于上次表现计算，显示加重建议标签 |
| 实际重量   | number | 用户输入 | 可覆盖建议值                       |
| 实际次数   | number | 用户输入 | 本组实际完成的次数                 |
| 目标次数   | number | 训练计划 | 用于判断是否达标                   |
| 倒计时剩余 | number | 计时器   | 组间休息剩余秒数                   |
| 已完成组数 | number | 实时计算 | x/总组数                           |

### States

| State              | Display                                | Trigger          |
| ------------------ | -------------------------------------- | ---------------- |
| 未开始             | 所有动作卡片折叠，显示「开始训练」按钮 | 刚进入页面       |
| 进行中（记录组）   | 当前动作展开，显示组输入区域           | 用户选中一个动作 |
| 进行中（组间休息） | 底部倒计时面板上滑，其他动作卡片变暗   | 一组记录完成     |
| 已完成             | 动作卡片标记 ✓，进度显示完成           | 所有组/动作完成  |
| 后台计时中         | 通知栏/锁屏显示倒计时剩余              | App 切到后台     |

### Validation Rules

- 重量必须 > 0
- 次数必须 ≥ 0（0 表示失败）
- 倒计时范围 30-600 秒

## UI Function 3: 训练计划管理

### Placement

- **Mode**: tab-page（Tab 2）
- **Target Page**: 训练计划页
- **Position**: 底部 Tab 栏进入

### Description

创建、编辑和管理训练计划。页面分为查看模式和编辑模式。支持按星期排期，定义每个训练日的动作。每个动作支持固定模式（统一组×次×重量）和自定义模式（逐组设置）。同时支持固定天数间隔模式，用户设置训练间隔天数（0-6天），系统按训练日顺序自动循环排期。

### User Interaction Flow

1. 进入计划页 → 默认显示查看模式，展示当前激活的训练计划
2. 查看模式展示：计划名称、每周各训练日卡片（显示训练类型、动作列表、下次训练日期）
3. 点击「新建计划」或进入编辑模式 → 切换为编辑界面
4. 编辑模式：输入计划名称 → 选择训练日（星期按钮：周一~周日开关）
   4a. 如选择「固定间隔」模式 → 输入间隔天数 → 系统自动循环排期
5. 每个训练日卡片内 → 从下拉列表选择动作
6. 每个动作选择模式：
   - 固定模式：输入组数、次数、重量（如 3组 x 8次 @ 60kg）
   - 自定义模式：逐组设置不同的重量和次数
7. 保存计划 → App 自动排期到日历

### Data Requirements

| Field        | Type   | Source   | Notes                                                      |
| ------------ | ------ | -------- | ---------------------------------------------------------- |
| 计划名称     | string | 用户输入 | 如「5/3/1 周期」                                           |
| 训练日排期   | array  | 用户选择 | 星期几训练（周一~周日开关）                                |
| 训练日列表   | array  | 用户编辑 | 每日包含：名称、动作列表                                   |
| 动作详情     | object | 动作库   | 动作名、模式（固定/自定义）、组数×次数×重量                |
| 下次训练日期 | date   | 系统计算 | 基于星期排期计算，显示在训练日卡片上                       |
| 排期方式     | string | 用户选择 | weekly_fixed（每周固定日）/ fixed_interval（固定天数间隔） |

### States

| State        | Display                                     | Trigger                |
| ------------ | ------------------------------------------- | ---------------------- |
| 查看模式     | 计划概览、各训练日卡片、下次训练日期        | 进入计划页             |
| 编辑模式     | 可编辑的计划表单、星期选择器、动作编辑器    | 点击「新建计划」或编辑 |
| 空计划       | 提示创建第一个计划                          | 首次使用               |
| 选择排期方式 | 显示排期方式切换控件（每周固定日/固定间隔） | 编辑模式中选择排期方式 |

### Validation Rules

- 计划名称不能为空
- 至少选择 1 个训练日（星期）
- 每个训练日至少包含 1 个动作
- 每个动作必须设置目标组数和次数

## UI Function 4: 动作库

### Placement

- **Mode**: push-page（动作库列表）/ push-page（动作详情）
- **Target Page**: 动作库页 / 动作详情页
- **Position**: 从设置页「动作库」入口进入，或在计划编辑时选择动作

### Description

内置常见力量举和辅助训练动作的分类列表，支持用户自定义动作。点击动作进入详情页，展示个人记录、进步曲线和最近训练历史。

### User Interaction Flow

1. 打开动作库 → 按分类展示动作列表（核心力量举、上肢推、上肢拉、下肢辅助、肩部等）
2. 点击动作 → 进入动作详情页
3. 动作详情页展示：
   - 基本信息：名称、分类、加重增量、默认休息时间、总训练次数
   - 个人记录：最高重量、最高容量（含达成日期）
   - 进步曲线：重量随时间变化的折线图，PR 点用绿色高亮标注
   - 最近 5 次训练：每次的组数×次数×重量，达标/未达标状态，容量和疲劳度
4. 从训练计划创建时 → 通过下拉选择器从动作库中勾选动作添加到训练日
5. 可在详情页点击「编辑」修改动作配置

### Data Requirements

| Field        | Type    | Source            | Notes                                          |
| ------------ | ------- | ----------------- | ---------------------------------------------- |
| 动作名称     | string  | 预置/用户输入     | 唯一标识                                       |
| 分类         | string  | 预置分类          | 核心力量举/上肢推/上肢拉/下肢/核心/肩部/自定义 |
| 加重增量     | number  | 预置默认/用户配置 | 单位 kg                                        |
| 默认休息时间 | number  | 预置默认/用户配置 | 单位秒                                         |
| 是否自定义   | boolean | 系统标记          | 区分预置和用户创建                             |
| 最高重量     | number  | 历史最大值        | PR 记录                                        |
| 最高容量     | number  | 历史最大值        | PR 记录                                        |
| 进步曲线数据 | array   | 训练记录          | [{date, weight}]                               |
| 最近训练记录 | array   | 训练记录          | 最近 5 次详情                                  |

### States

| State    | Display                             | Trigger              |
| -------- | ----------------------------------- | -------------------- |
| 浏览模式 | 分类列表                            | 从设置进入           |
| 动作详情 | 基本信息 + PR + 进步曲线 + 最近训练 | 点击某个动作         |
| 选择模式 | 下拉选择器，按分类列出动作          | 从计划编辑中选择动作 |

### Validation Rules

- 动作名称不可重复
- 加重增量 > 0
- 休息时间 30-600 秒

## UI Function 5: 历史数据与进步分析

### Placement

- **Mode**: tab-page（Tab 3）
- **Target Page**: 历史分析页
- **Position**: 底部 Tab 栏进入

### Description

训练记录和历史分析。页面顶部有 4 段切换控件（历史/进步/容量/PR），分别展示不同视角的训练数据。

### User Interaction Flow

1. 进入历史页面 → 默认显示「历史」面板，按日期倒序展示训练记录卡片
2. 训练记录卡片展示：日期、训练类型标签、动作详情（动作名 + 重量x次数x组数）、总容量、自评评分
3. 如有 PR，动作行显示绿色 PR 徽章
4. 切换到「进步」面板 → 选择动作（下拉选择器）→ 显示该动作重量随时间变化的折线图，PR 点绿色高亮
5. 切换到「容量」面板 → 显示近 6 周总容量柱状图 + 汇总卡片（本周容量、上周对比、月累计）
6. 切换到「PR」面板 → 显示各动作的个人最佳记录卡片（动作名、最高重量、达成日期、当日容量）

### Data Requirements

| Field    | Type   | Source             | Notes                 |
| -------- | ------ | ------------------ | --------------------- |
| 训练日期 | date   | 训练记录           | 记录卡片和图表 X 轴   |
| 动作详情 | string | 训练记录           | 如"卧推 60kg x 5 x 5" |
| 总容量   | number | 计算得出           | Σ(重量×次数)          |
| 自评评分 | number | 训练感受           | 1-10                  |
| 动作重量 | number | 训练记录           | 进步曲线 Y 轴         |
| 每周容量 | number | 计算得出           | 容量柱状图            |
| PR 值    | number | 历史最大值         | 最高重量、最高容量    |
| PR 日期  | date   | 达成 PR 的训练日期 | PR 记录               |

### States

| State    | Display                       | Trigger        |
| -------- | ----------------------------- | -------------- |
| 无数据   | 提示「完成你的第一次训练」    | 首次使用       |
| 历史列表 | 训练记录卡片列表，按日期倒序  | 默认面板       |
| 进步曲线 | 动作选择器 + 折线图 + PR 标注 | 选择具体动作后 |
| 容量趋势 | 柱状图 + 汇总卡片             | 切换到容量面板 |
| PR 记录  | 各动作最佳记录卡片            | 切换到 PR 面板 |

### Validation Rules

- 进步曲线需要至少 2 个数据点
- PR 判断：单次重量超过历史最高即为新 PR

## UI Function 6: 身体数据记录

### Placement

- **Mode**: push-page
- **Target Page**: 身体数据页
- **Position**: 从设置页或其他入口进入（非 Tab 页）

### Description

记录体重和围度数据，展示变化趋势图。可关联训练周期查看身体变化。

### User Interaction Flow

1. 进入身体数据页 → 显示「记录数据」按钮 + 最近一次身体数据
2. 点击「记录数据」 → 输入日期、体重、各围度 → 保存
3. 点击「趋势图」 → 选择指标 → 显示随时间变化的折线图

### Data Requirements

| Field    | Type   | Source   | Notes                     |
| -------- | ------ | -------- | ------------------------- |
| 记录日期 | date   | 用户选择 | 默认今天                  |
| 体重     | number | 用户输入 | 精确到 0.1kg              |
| 胸围     | number | 用户输入 | 精确到 0.1cm，可选        |
| 腰围     | number | 用户输入 | 精确到 0.1cm，可选        |
| 臂围     | number | 用户输入 | 精确到 0.1cm，可选        |
| 大腿围   | number | 用户输入 | 精确到 0.1cm，可选        |
| 备注     | string | 用户输入 | 自由文本，最大200字，可选 |

### States

| State    | Display                        | Trigger          |
| -------- | ------------------------------ | ---------------- |
| 无数据   | 提示「记录你的第一次身体数据」 | 首次使用         |
| 录入模式 | 各项输入字段                   | 点击「记录数据」 |
| 趋势展示 | 折线图 + 数据列表              | 点击「趋势图」   |

### Validation Rules

- 体重为必填项，围度为选填
- 所有数值 > 0

## UI Function 7: 其他运动记录

### Placement

- **Mode**: push-page
- **Target Page**: 其他运动记录页
- **Position**: 从日历页进入，左上角返回日历

### Description

记录非力量训练的运动数据。预设 4 种运动类型（游泳/跑步/骑行/瑜伽），支持自定义运动类型。

### User Interaction Flow

1. 进入页面 → 显示运动类型选择网格（2x2 卡片：游泳、跑步、骑行、瑜伽）
2. 点击运动卡片 → 选中高亮，下方显示录入表单
3. 表单包含：距离、时间（分:秒）、趟数/次数、配速、心率、卡路里、备注。用户可在自定义运动时选择需要记录的指标。
4. 点击「自定义运动」 → 创建新运动类型
5. 点击「保存」 → 保存运动记录，返回日历

### Data Requirements

| Field     | Type   | Source        | Notes                                |
| --------- | ------ | ------------- | ------------------------------------ |
| 运动类型  | string | 预设/自定义   | 游泳/跑步/骑行/瑜伽/自定义           |
| 距离      | number | 用户输入      | 单位 m 或 km                         |
| 时间      | number | 用户输入      | 分:秒                                |
| 趟数/次数 | number | 用户输入      | 可选                                 |
| 备注      | string | 用户输入      | 可选                                 |
| 日期      | date   | 日历          | 记录日期                             |
| 配速      | string | 计算/用户输入 | 根据距离和时间自动计算，也可手动输入 |
| 心率      | number | 用户输入      | 单位 bpm，可选                       |
| 卡路里    | number | 用户输入/计算 | 单位 kcal，可选                      |

### States

| State        | Display                           | Trigger    |
| ------------ | --------------------------------- | ---------- |
| 选择运动类型 | 2x2 卡片网格 + 「自定义运动」按钮 | 进入页面   |
| 录入数据     | 距离、时间、趟数、备注输入字段    | 选择类型后 |

### Validation Rules

- 运动类型名称不能为空
- 数值型指标必须 > 0

## UI Function 8: 训练感受记录

### Placement

- **Mode**: push-page（自动进入）
- **Target Page**: 训练感受页
- **Position**: 训练完成后自动进入，无返回按钮

### Description

训练结束后的感受记录页面。记录整体疲劳度和满意度，以及各动作的独立感受。可跳过。

### User Interaction Flow

1. 训练完成 → 自动进入感受记录页，顶部显示完成图标和训练摘要
2. 滑动设置疲劳度（1-10），滑块从绿到黄到红
3. 滑动设置满意度（1-10），滑块从红到黄到绿
4. 为每个动作写感受备注（可选，每个动作独立文本框）
5. 写整体备注（可选）
6. 点击「保存」→ 保存感受数据 → 返回日历
7. 点击「跳过」→ 跳过感受记录 → 返回日历

### Data Requirements

| Field      | Type         | Source   | Notes                     |
| ---------- | ------------ | -------- | ------------------------- |
| 疲劳度     | number(1-10) | 滑块     | 1=轻松, 10=筋疲力尽       |
| 满意度     | number(1-10) | 滑块     | 1=很差, 10=完美           |
| 各动作感受 | string       | 文本输入 | 每个动作独立备注，可选    |
| 整体备注   | string       | 文本输入 | 自由文本，可选            |
| 训练摘要   | string       | 训练记录 | 如"推日 · 总容量 8,400kg" |

### States

| State    | Display                              | Trigger        |
| -------- | ------------------------------------ | -------------- |
| 默认     | 疲劳度默认 6，满意度默认 7，备注为空 | 训练完成后进入 |
| 保存确认 | Toast 提示「感受已保存」             | 点击保存后     |

### Validation Rules

- 疲劳度和满意度有默认值，用户可直接保存

## UI Function 9: 统计概览

### Placement

- **Mode**: tab-page（Tab 4）
- **Target Page**: 统计页
- **Position**: 底部 Tab 栏进入

### Description

宏观训练概览仪表盘。汇总本周/本月的训练量、频率和个人记录，帮助用户从全局视角掌握训练状态。

### User Interaction Flow

1. 进入统计页 → 显示本周训练概览
2. Hero 卡片：本周训练总容量 + 周环比变化百分比（上升绿色/下降红色）
3. 四宫格汇总卡片：
   - 本周训练次数（及目标次数）
   - 本月训练次数（及连续周数）
   - 总训练时长（本周累计）
   - 本月新增个人记录数
4. 周训练容量柱状图：近 8 周的容量对比，本周高亮，其他周灰色
5. 个人记录列表：各动作 1RM 估测及达成日期，点击「查看全部」跳转历史页 PR 面板
6. 训练频率热力图：近 4 周的活动方块图（类似 GitHub 贡献图），深浅表示训练强度，计划中日期用蓝色边框标注

### Data Requirements

| Field        | Type   | Source   | Notes                    |
| ------------ | ------ | -------- | ------------------------ |
| 本周容量     | number | 计算得出 | Σ(重量×次数)             |
| 周环比变化   | number | 对比上周 | 百分比，正为升、负为降   |
| 本周训练次数 | number | 训练记录 |                          |
| 本月训练次数 | number | 训练记录 |                          |
| 连续训练周数 | number | 计算得出 |                          |
| 本周训练时长 | number | 训练记录 | 小时                     |
| 本月新增 PR  | number | PR 记录  |                          |
| 周容量数据   | array  | 计算得出 | 近 8 周 [{week, volume}] |
| 个人记录     | array  | PR 记录  | 各动作 1RM 估测          |
| 训练频率     | array  | 训练记录 | 近 28 天每日训练强度     |

### States

| State  | Display            | Trigger    |
| ------ | ------------------ | ---------- |
| 无数据 | 提示完成第一次训练 | 首次使用   |
| 有数据 | 完整仪表盘         | 有训练记录 |

### Validation Rules

- 周环比计算：本周容量 / 上周容量 - 1
- 热力图强度分级：休息=0.1，轻度=0.4-0.6，中度=0.7-0.8，重度=0.9+

## UI Function 10: 设置

### Placement

- **Mode**: tab-page（Tab 5）
- **Target Page**: 设置页
- **Position**: 底部 Tab 栏进入

### Description

应用设置和偏好管理。包含训练偏好配置、动作库入口和数据管理功能。

### User Interaction Flow

1. 进入设置页 → 显示设置分组列表
2. 训练偏好：重量单位切换（kg/lbs）、默认休息时间选择
3. 点击「动作库」→ 进入动作库页面
4. 数据管理：导出数据（选择范围：全部/3个月/6个月）、导入数据、清除所有数据
5. 清除和导入操作需二次确认（底部弹出确认面板）

### Data Requirements

| Field        | Type   | Source   | Notes                 |
| ------------ | ------ | -------- | --------------------- |
| 重量单位     | string | 用户选择 | kg / lbs              |
| 默认休息时间 | number | 用户选择 | 90/120/180/240/300 秒 |
| 导出范围     | string | 用户选择 | all / 3m / 6m         |

### States

| State    | Display          | Trigger              |
| -------- | ---------------- | -------------------- |
| 默认     | 设置列表         | 进入页面             |
| 导出确认 | 底部弹出选择面板 | 点击「导出数据」     |
| 清除确认 | 底部弹出警告面板 | 点击「清除所有数据」 |
| 导入确认 | 底部弹出确认面板 | 点击「导入数据」     |

### Validation Rules

- 清除数据操作不可撤销，必须二次确认
