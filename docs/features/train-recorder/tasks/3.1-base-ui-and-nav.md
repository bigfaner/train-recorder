# 3.1: Base UI Components + Navigation Layout

## Description

Implement the base UI component library and the full navigation structure (root layout, tab bar, stack navigator) for the app.

## Reference Files

- ui/ui-design.md — Design System section (colors, typography, components specs)
- design/tech-design.md — Directory Structure (app/ routing files)
- prd/prd-ui-functions.md — Navigation Structure section, Page Composition table
- ui/prototype/index.html — Page inventory and navigation prototype

## Acceptance Criteria

- [ ] Base components: Button (primary/secondary/destructive), Card, Input, Slider, Tag/Pill, TimerDisplay
- [ ] All components match ui-design.md specs (colors, sizes, border radius, typography)
- [ ] app/_layout.tsx root layout wraps app in providers
- [ ] app/(tabs)/_layout.tsx implements 5-tab bottom navigation (日历/计划/历史/统计/设置)
- [ ] Tab bar matches ui-design.md: 5 icons, active=accent, inactive=tertiary, height 83px
- [ ] Stack navigator configured for all push pages (workout, feeling, other-sport, exercise-library, exercise-detail, body-data, plan-editor, training-day-editor, onboarding, sport-editor)
- [ ] Navigation between tab pages and push pages works without errors
- [ ] Safe area insets respected on iOS and Android
- [ ] Placeholder screens for all routes render correctly

## User Stories

No direct user story mapping. UI infrastructure for all subsequent page tasks.

## Implementation Notes

- Use Expo Router v4 file-based routing
- Tab icons: calendar, clipboard.list, chart.bar, figure.stand, gearshape (SF Symbols / Material equivalents)
- Colors from ui-design.md: Accent=#0071e3, Success=#30d158, Error=#ff3b30, etc.
- Typography fallback: -apple-system, 'Segoe UI', Roboto, sans-serif
- Components in src/components/ui/ directory
- All push pages hide tab bar (header presentation: modal or card)
