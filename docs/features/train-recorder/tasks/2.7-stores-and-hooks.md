# 2.7: Zustand Stores + Custom Hooks

## Description

Implement the three Zustand stores (workout, timer, settings) and their associated custom hooks that bridge the service layer to the UI layer.

## Reference Files

- design/tech-design.md — State Layer section (workoutStore, timerStore, settingsStore)
- design/tech-design.md — Zustand testing pattern in Testing Strategy section

## Acceptance Criteria

- [ ] workoutStore manages active workout session state: current session, exercises, sets, loading/error states
- [ ] workoutStore actions: startWorkout, selectExercise, recordSet, completeExercise, completeWorkout, exitWorkout (partial), restoreSession
- [ ] timerStore wraps TimerService: isActive, remainingSeconds, start/pause/resume/skip/adjust actions
- [ ] settingsStore caches user preferences: weightUnit, defaultRestTime, notifications settings
- [ ] useWorkout hook exposes workoutStore actions + computed progress (completed/total exercises and sets)
- [ ] useTimer hook exposes timerStore state + formatted display time
- [ ] useCalendar hook wraps CalendarComputer for month/day computation with plan data
- [ ] useProgressiveOverload hook provides suggestion data for the current exercise
- [ ] useExerciseHistory hook provides summary data for exercise detail views
- [ ] useUnitConversion hook provides conversion functions reactive to settings changes
- [ ] Unit tests for all store state transitions and async actions

## User Stories

- US-2 AC1-AC9: Workout execution state management (start, record, timer, exit, restore)
- US-10 AC1-AC2: Mid-workout exit with partial completion state
- US-11 AC3,AC6: Session restore after background/force-close

## Implementation Notes

- Zustand testing: access state via `useStore.getState()` in Jest; call actions directly
- workoutStore.startWorkout: creates WorkoutSession + WorkoutExercises from plan data
- workoutStore.recordSet: creates WorkoutSet, triggers ProgressiveOverload.recordResult, starts TimerService
- workoutStore.exitWorkout: saves completed data, marks session completed_partial, persists state for recovery
- Stores are singleton instances; hooks wrap store.subscribe + store.getState
