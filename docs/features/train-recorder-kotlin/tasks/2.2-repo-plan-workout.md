# Task 2.2: Plan & Workout Repository Implementations

## Description

Implement `TrainingPlanRepository` and `WorkoutRepository`.

### TrainingPlanRepository
- Create, update, and delete plans with nested training days and exercises
- Activate a plan (automatically deactivates all other plans)
- Cascade operations for nested structures
- Application-level cascade deletes (no SQL foreign keys)

### WorkoutRepository
- Create a workout session from a training plan
- Record sets within a workout exercise
- Update exercise status and session status
- Support partial completion (mark session as partially completed)
- Delete a session with cascade (remove all child exercise sets)
- Backfill past sessions (creates session with `is_backfill = 1`)

### SQLDelight Query Files
- Create `.sq` query files for both repositories covering all operations.
