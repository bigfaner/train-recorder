# 1.3: Database Schema + Migration Manager

## Description

Implement the database initialization module with all 15 CREATE TABLE statements and indexes from schema.sql, plus a migration manager for schema version management.

## Reference Files

- design/schema.sql — Full DDL (15 tables + 15 indexes)
- design/er-diagram.md — Entity relationships and Index Design table
- design/tech-design.md — Data Layer section

## Acceptance Criteria

- [ ] src/db/database.ts initializes expo-sqlite database connection
- [ ] src/db/schema.ts contains all 15 CREATE TABLE statements matching schema.sql exactly
- [ ] All 15 CREATE INDEX statements from schema.sql executed
- [ ] src/db/migrations/ directory with migration manager that tracks schema version
- [ ] Database initializes without errors on first run
- [ ] Re-running initialization is idempotent (no errors, no data loss)
- [ ] All column types, constraints, and defaults match er-diagram.md entity details

## User Stories

No direct user story mapping. Foundational infrastructure.

## Implementation Notes

- Use expo-sqlite v15 API (openDatabaseSync or withDatabaseSync)
- Store migration version in user_settings table (setting_key = 'schema_version')
- All tables use INTEGER PRIMARY KEY AUTOINCREMENT + INTEGER NOT NULL UNIQUE for biz_key
- No foreign key constraints (application-layer integrity per design decision)
- Seed exercises table with built-in exercises from PRD §5.5 on first init
