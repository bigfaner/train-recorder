# 2.1: ProgressiveOverload Service

## Description

Implement the progressive overload suggestion algorithm that calculates weight recommendations for each exercise based on historical performance.

## Reference Files

- design/tech-design.md — ProgressiveOverload interface, PRD Coverage Map (US-3, US-10, US-12, US-13, US-16, US-17)
- prd/prd-spec.md §5.3 — Progressive overload rules table
- src/db/repositories/ — WorkoutExercise, WorkoutSet, Exercise repos from Phase 1

## Acceptance Criteria

- [ ] calculateSuggestion(exerciseBizKey, targetReps) returns OverloadSuggestion
- [ ] All sets met → direction='increase', suggestedWeight = previousWeight + exercise.increment
- [ ] Some sets missed → direction='maintain', suggestedWeight = previousWeight
- [ ] Consecutive 2 sessions missed → direction='decrease', suggestedWeight = previousWeight * 0.9
- [ ] Consecutive 3 sessions completed → UI hint "考虑加大增量？" (no auto-modify)
- [ ] No history → suggestedWeight = null, reason explains first-time input needed
- [ ] Decrease value rounded to nearest plate combination via UnitConversion.roundToPlate
- [ ] recordResult(exerciseBizKey, sets) updates consecutive counts
- [ ] recalculateChain(exerciseBizKey, fromDate) rebuilds suggestion chain from a given date
- [ ] Unit tests: 90%+ coverage for all overload scenarios

## User Stories

- US-3 AC1-AC8: All progressive overload acceptance criteria
- US-10 AC4: Completed exercises participate in overload, incomplete do not
- US-12 AC2: Edit workout triggers chain recalculation
- US-13 AC4: Backlog workout triggers chain recalculation
- US-16 AC3: Skipped exercises excluded from overload
- US-17 AC3: Same exercise appears multiple times with independent overload chains

## Implementation Notes

- Calculate per WorkoutExercise.biz_key (not per Exercise) to handle same-exercise-multiple-times (US-17)
- Query WorkoutSets for a given exercise_biz_key ordered by session_date, check is_target_met
- Consecutive counts: iterate backwards from most recent session, count consecutive met/missed
- recalculateChain: from the given date forward, re-evaluate all suggestions for subsequent sessions
- Use exercise.increment from Exercise repo for weight increase amounts
