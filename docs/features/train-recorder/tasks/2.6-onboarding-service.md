# 2.6: Onboarding Service + Plan Templates

## Description

Implement the onboarding service with plan templates for first-time users, including template selection, plan creation from templates, and onboarding state management.

## Reference Files

- design/tech-design.md — OnboardingService interface, PlanTemplate type
- prd/prd-spec.md §5.11.3 — First-time onboarding flow and template specs

## Acceptance Criteria

- [ ] getTemplates() returns at least 3 templates: PPL (推拉蹲), Upper/Lower (上下肢), Full Body (全身)
- [ ] Each template has: name, description, plan_mode, schedule_mode, days with exercises and sets_config
- [ ] createPlanFromTemplate(template) creates TrainingPlan + TrainingDays + PlanExercises from template
- [ ] isOnboardingComplete() reads user_settings key 'onboarding_completed'
- [ ] markOnboardingComplete() sets user_settings key to 'true'
- [ ] resetOnboarding() clears the flag for re-viewing from Settings
- [ ] Template exercises reference built-in exercise names from seed data
- [ ] Unit tests for template creation and state management

## User Stories

- US-18 AC1-AC4: Welcome flow, template selection, pre-filled exercises, settings re-view

## Implementation Notes

- Templates are static data (hardcoded array), not database records
- PPL template: 推日 (卧推/上斜卧推/哑铃飞鸟/三头下压), 拉日 (硬拉/划船/引体/二头弯举), 蹲日 (深蹲/前蹲/腿举/腿弯举)
- createPlanFromTemplate: batch generate snowflake IDs for plan + all days + all exercises
- Onboarding state persisted in user_settings table with key 'onboarding_completed'
