# 1.2: TypeScript Types + Snowflake ID Generator

## Description

Define all TypeScript types and interfaces for the 15-entity data model, and implement the Snowflake ID generator for generating globally unique bigint biz_keys.

## Reference Files

- design/tech-design.md — Interfaces section (BaseRepository, all service interfaces), Data Models Field Quick Reference, Cross-Layer Data Map
- design/er-diagram.md — All 15 entity field definitions
- design/schema.sql — Column types and constraints

## Acceptance Criteria

- [ ] src/types/index.ts exports types for all 15 entities matching er-diagram.md field definitions
- [ ] SetsConfig discriminated union type defined (mode: 'fixed' | 'custom')
- [ ] All service interfaces from tech-design defined (CalendarComputer, ProgressiveOverload, TimerService, PRTracker, ExerciseHistoryService, UnitConversion, DataExportService, DataImportService, OnboardingService)
- [ ] Error types defined with codes matching tech-design Error Handling section
- [ ] src/services/snowflake.ts implements SnowflakeIdGenerator interface
- [ ] generate() produces unique bigint values; generateBatch(n) produces n unique values
- [ ] Unit tests for snowflake ID uniqueness and batch generation pass

## User Stories

No direct user story mapping. Foundational infrastructure.

## Implementation Notes

- Use bigint for all biz_key fields; SQLite stores as INTEGER (64-bit)
- SetsConfig: `{ mode: 'fixed', target_reps, target_weight, target_repeat } | { mode: 'custom', sets: Array<{target_reps, target_weight}> }`
- Snowflake: 41-bit timestamp + 10-bit machine-id + 12-bit sequence; machine-id derived from device fingerprint
- Export both entity types and service interface types
