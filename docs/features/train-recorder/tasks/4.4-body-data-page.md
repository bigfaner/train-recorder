# 4.4: Body Data Page

## Description

Implement the body measurement recording page with data entry form, trend chart, and history list.

## Reference Files

- ui/ui-design.md — Component 6: 身体数据记录
- ui/prototype/body-data.html — Interactive prototype
- prd/prd-ui-functions.md — UF-6: 身体数据记录
- prd/prd-spec.md §5.7 — Body measurement specs

## Acceptance Criteria

- [ ] Latest data card at top: most recent weight + change trend arrow (green down / red up) + date
- [ ] "+ 记录数据" button opens entry form (push page)
- [ ] Entry form: date picker (default today), weight (required), chest/waist/arm/thigh circumference (optional), note
- [ ] Save creates BodyMeasurement record
- [ ] Segment selector: 趋势图 / 历史记录
- [ ] Trend chart: line chart of selected metric over time (victory-native)
- [ ] Metric selector: weight/chest/waist/arm/thigh
- [ ] History list: records sorted by date descending with edit/delete on swipe
- [ ] Edit pre-fills form with existing data
- [ ] Empty state: "记录你的第一次身体数据" prompt
- [ ] Nullable circumference fields: only save non-null values

## User Stories

- US-8 AC1-AC5: Body data recording, trend chart, partial fill, backlog date, edit support

## Implementation Notes

- Data from BodyMeasurement repo
- Trend chart from victory-native LineChart
- Change trend: compare latest vs previous record
- Date picker for historical data entry (cannot exceed today)
- Circumference fields optional: validation allows null
