# 4.2: Stats Page (Tab 4)

## Description

Implement the statistics overview dashboard with hero card, four-grid summary, weekly volume bar chart, PR list, and training frequency heatmap.

## Reference Files

- ui/ui-design.md — (not a standalone component, but design system applies)
- ui/prototype/stats.html — Interactive prototype
- prd/prd-ui-functions.md — UF-9: 统计概览
- prd/prd-spec.md §5.10 — Statistics overview specs with sample data and validation rules

## Acceptance Criteria

- [ ] Hero card: weekly volume with week-over-week change percentage (green up / red down)
- [ ] Four-grid summary cards:
  - Card 1: Weekly sessions (current/target)
  - Card 2: Monthly sessions (consecutive weeks)
  - Card 3: Weekly total duration (hours)
  - Card 4: Monthly new PRs
- [ ] Weekly volume bar chart: last 8 weeks, current week highlighted in accent, last week green border
- [ ] PR list: top 4 exercises with estimated 1RM and date, "查看全部" link to history PR panel
- [ ] Training frequency heatmap: 28-day grid, intensity shading (rest=0.1, light=0.4-0.6, moderate=0.7-0.8, heavy=0.9+)
- [ ] Planned but incomplete dates: blue border on heatmap
- [ ] Empty state: "完成你的第一次训练" prompt
- [ ] Week-over-week formula: thisWeekVolume / lastWeekVolume - 1; no data shows "--"
- [ ] 1RM estimation formula: weight × (1 + reps / 30)

## User Stories

No direct user story mapping. PRD §5.10 statistics overview feature.

## Implementation Notes

- All data computed from WorkoutSession + WorkoutSet + PersonalRecord repos
- Hero card weekly volume: sum of (weight × reps) for all sets this week
- Four-grid: count sessions in current week/month, compute duration from started_at/ended_at
- Bar chart: victory-native BarChart with custom bar colors
- Heatmap: custom grid component with intensity-based coloring
- PRTracker.getEstimated1RM for 1RM calculations
