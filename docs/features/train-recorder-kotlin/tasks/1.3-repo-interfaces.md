# Repository Interfaces

## Description

Define all 11 repository interfaces in `shared/commonMain`. All suspend functions return `Result<T>`. All Flow-based queries return unwrapped `Flow`. Include `WeightSuggester` use case class.

### Interfaces (11)

1. **ExerciseRepository** — CRUD for exercises, query by category/muscle group
2. **TrainingPlanRepository** — CRUD for training plans and their days/exercises/set configs
3. **WorkoutRepository** — Workout session lifecycle (create, start, pause, complete, abandon), exercise and set recording
4. **WeightSuggestionRepository** — Store and retrieve weight suggestions per exercise
5. **PersonalRecordRepository** — Track and query personal records, detect new PRs
6. **ScheduleCalculator** — Calculate next training day based on plan mode (infinite_loop / fixed_cycle) and schedule mode (weekly_fixed / fixed_interval)
7. **BodyDataRepository** — Body measurements CRUD and historical queries
8. **OtherSportRepository** — Other sport types, metrics, records, and metric values
9. **TimerService** — Timer state management (start, pause, resume, reset, tick events)
10. **SettingsRepository** — User settings read/write (weight unit, export format, etc.)
11. **FeelingRepository** — Workout and exercise feeling CRUD

### Use Case Class

- **WeightSuggester** — Encapsulates weight suggestion logic using historical data, personal records, and suggestion hints
