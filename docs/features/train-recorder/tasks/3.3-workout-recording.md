# 3.3: Workout Execution - Recording UI

## Description

Implement the core workout execution page for exercise card display, set recording, and workout completion flow (excluding timer UI which is task 3.4).

## Reference Files

- ui/ui-design.md — Component 2: 训练执行 (layout, states, interactions, data binding)
- ui/prototype/workout.html — Interactive prototype
- prd/prd-ui-functions.md — UF-2: 训练执行
- prd/prd-spec.md §5.2, §5.2.1-§5.2.3 — Workout execution, mid-exit, order adjust, same exercise

## Acceptance Criteria

- [ ] Exercise cards display: name, suggested weight (with +increment label), target sets×reps
- [ ] Current exercise card expanded with weight/reps input and "完成本组" button
- [ ] Completed exercise cards collapsed with ✓ and actual weight×reps summary
- [ ] Pending exercise cards shown in muted grey
- [ ] Weight input pre-filled with suggested weight, user can override
- [ ] Override shows "自定义" label instead of suggestion label
- [ ] "完成本组" saves set data via workoutStore.recordSet
- [ ] Progress indicator: completed/total exercises
- [ ] Back button shows confirmation dialog: "已完成 x/y 动作，确定结束？"
- [ ] Confirm exit saves partial data as completed_partial
- [ ] Long-press to drag-reorder exercises
- [ ] Left-swipe to show "跳过" option on exercises
- [ ] Same exercise multiple times shown with note distinction (e.g., "深蹲 #1", "深蹲 #2 - 暂停深蹲")
- [ ] All exercises completed → auto-navigate to feeling page
- [ ] "加一组" button for extra sets beyond planned

## User Stories

- US-2 AC1-AC8: Full workout recording flow (suggestions, recording, extra sets, partial exit)
- US-10 AC1-AC3: Mid-workout exit with confirmation and partial save
- US-16 AC1-AC4: Drag reorder, swipe skip, unskip
- US-17 AC1-AC3: Same exercise multiple times with independent tracking

## Implementation Notes

- Tab bar hidden during workout (full-screen immersive mode)
- Workout data from workoutStore; suggestions from useProgressiveOverload hook
- Large font for weight display (48px SF Pro Display Bold per ui-design.md)
- Single-tap "完成本组" for ≤2 clicks per set (US-2 AC6)
- After "完成本组": timer starts (handled by 3.4, but this task should trigger the action)
