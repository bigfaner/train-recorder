# 3.5: Post-Workout Feeling Page

## Description

Implement the post-workout feeling recording page with fatigue/satisfaction sliders, per-exercise notes, and overall note input.

## Reference Files

- ui/ui-design.md — Component 8: 训练感受记录 (layout, states, interactions, data binding)
- ui/prototype/feeling.html — Interactive prototype
- prd/prd-ui-functions.md — UF-8: 训练感受记录
- prd/prd-spec.md §5.4 — Training feeling specs

## Acceptance Criteria

- [ ] Page auto-enters after workout completion (no back button to workout page)
- [ ] Training summary at top: training type + total volume (e.g., "推日 · 总容量 8,400kg")
- [ ] Fatigue slider (1-10) with gradient label (轻松 → 筋疲力尽), default=6
- [ ] Satisfaction slider (1-10) with gradient label (很差 → 完美), default=7
- [ ] Per-exercise note cards: shows exercise name + weight×sets×reps + text input
- [ ] Only completed exercises shown (skipped excluded)
- [ ] Overall note text input
- [ ] "保存" button saves and returns to calendar
- [ ] "跳过" button skips with defaults and returns to calendar
- [ ] High fatigue (≥8) + low satisfaction (≤4) → orange warning labels + "建议下次降低强度"
- [ ] Save triggers ProgressiveOverload.recordResult for all exercises

## User Stories

- US-5 AC1-AC5: Feeling recording with sliders, per-exercise notes, defaults, edit support
- US-3: Save triggers overload update for next session suggestions

## Implementation Notes

- Auto-enter via navigation.replace after workout completion (user cannot go back to workout)
- Sliders use custom Slider component from base UI (src/components/ui/Slider)
- Feeling data saved via workoutStore actions → Feeling + ExerciseFeeling repos
- Only show exercises with exercise_status='completed' for per-exercise notes
