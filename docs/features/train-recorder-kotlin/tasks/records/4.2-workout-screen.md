### Key Decisions
- Separated pure logic (WorkoutHelper) from Compose UI (WorkoutScreen) for testability following CalendarScreen/CalendarHelper pattern
- Used AnimatedVisibility with expandVertically/shrinkVertically for exercise card expand/collapse animation
- Timer panel uses slideInVertically/slideOutVertically for slide-up animation after set completion
- Swipe-to-skip uses detectHorizontalDragGestures with offset threshold of 150px and confirmation dialog
- Long-press-to-reorder uses detectTapGestures onLongPress to select, then detectDragGestures on reorder handles for drag
- Exit confirmation dialog shows completed exercises count and completed sets count via ExitDialogState
- Progress indicator uses LinearProgressIndicator in top bar, consuming WorkoutProgress.fraction from WorkoutViewModel
- Weight input pre-fills from suggestedWeight in WorkoutExerciseUi, reps input pre-fills from targetReps
- Tab bar hidden by design - WorkoutRoute is not in TAB_ROUTES set, so TrainRecorderScaffold hides tab bar
